# Hacker Bob v1.3.2 Release Notes

## Highlights

`v1.3.2` fixes surface-discovery agent compatibility with the session read/write guards. Normal and deep surface discovery now keep bulky collection captures in temporary scratch outside the Bob session directory while continuing to produce compact artifacts for routing, summaries, and later phases.

## Surface Discovery Guard Compatibility

- Replaced session-local temp files such as `subdomains.tmp` with `mktemp` scratch outside `~/bounty-agent-sessions`.
- Removed `family_raw.txt`, `js_raw.txt`, and deep surface discovery `[SESSION]/raw/*` capture paths from the generated surface-discovery contracts.
- Kept compact derived artifacts in the session directory, including `attack_surface.json`, `surface-discovery-summary.json`, `deep-summary.json`, and bounded `.txt` summaries.
- Preserved the seven-step surface-discovery contract and the MCP-free surface-discovery agent boundary.

## Session Guard Boundaries

- Allowed compact surface-discovery summary reads for `surface-discovery-summary.json`, `deep-summary.json`, and `surface-leads.json`.
- Allowed direct writes only for compact surface-discovery-owned summaries such as `surface-discovery-summary.json` and `deep-summary.json`.
- Kept `surface-leads.json` write-protected so normal lead handling remains MCP-owned.
- Kept raw/proof/body/dump-style session reads blocked.

## Verification

- `npm run test:hooks` passes.
- `npm run test:prompts` passes.
- Rendered normal and deep surface-discovery Bash blocks pass both session guards with a concrete session path.
- `npm run release:check` passes.
