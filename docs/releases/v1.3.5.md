# Hacker Bob v1.3.5 Release Notes

`v1.3.5` hardens the shared MCP runtime contracts and release gates used by Claude, Codex, and generic MCP installs.

## What Changed

- Extracted implicit runtime behavior into explicit contracts and stores for findings, session state, verification rounds, wave handoffs, grade verdicts, pipeline events, pipeline session artifacts, handoff signing, and session authority.
- Rewired large runtime owner modules around those contracts while preserving their public MCP tool behavior.
- Centralized smart-contract egress policy and direct public HTTPS enforcement across chain clients, source fetches, and harness handoff paths.
- Tightened transport, tool registry, telemetry, session read/write guards, and secret screening around the shared authority model.
- Added deterministic package/release checks: authority inventory freshness, clean release audit, package surface documentation, dependency freshness checks, and stricter package policy tests.
- Regenerated Claude, Codex, generic MCP, role prompt, skill, and documentation surfaces to reflect the shared runtime behavior.
- Clarified Claude statusline update hints and suppress stale cached downgrade prompts after local installs.
- Removed obsolete replay/benchmark scripts and the stale generated offline-guide PDF from source control.

## Upgrade

```sh
npx -y hacker-bob@1.3.5 install /path/to/your/project
npx -y hacker-bob-cc@1.3.5 install /path/to/your/project
npx -y hacker-bob-codex@1.3.5 install /path/to/your/project
```
