# Hacker Bob v1.3.5 Release Notes

`v1.3.5` is a major feature release shipping OSS repo hunting mode (`/bob-oss`), the Kimi CLI adapter, CVE feed matching, installer symlink hardening, and a full overhaul of the shared MCP runtime contracts and release gates.

## Highlights

### OSS repo hunting mode

`/bob-oss <github-url>` (Claude) / `$bob-oss <github-url>` (Codex) runs the full Bob hunter pipeline against an open-source repository in a sandboxed Docker environment. Bob clones the repo, builds a toolchain image with ASAN/libFuzzer/Valgrind support, runs static scanning (CodeQL/Semgrep), and dispatches hunters against the `native` surface. Findings carry a `reachability` object with `caller_path`, `trigger_conditions`, and `severity_ceiling` so the grader can apply accurate severity ratings for CLI-local vs. daemon-reachable code paths.

A local OSS session dashboard (`/bob-status` → `oss-dashboard.html`) shows active repo sessions and per-session phase/finding counts without requiring a live MCP connection.

### Kimi CLI adapter

A third adapter (`--adapter kimi`) brings Hacker Bob to the Kimi CLI, matching the Codex adapter shape. A thin `hacker-bob-kimi` wrapper package (`npx hacker-bob-kimi install <dir>`) handles install alongside the existing `hacker-bob-cc` (Claude Code) and `hacker-bob-codex` wrappers.

### CVE feed matching

`bounty_public_intel` now cross-references NVD/GHSA feeds against the session's target domain and known surface endpoints, surfacing relevant recent CVEs as ranked leads in the hunter brief.

### Installer symlink hardening

`scripts/install.js` resolves symlinks before writing any Bob-owned file, closing a TOCTOU window where a malicious symlink in `.claude/` could redirect an install write to an arbitrary path.

## What Changed

- Added `/bob-oss` and `$bob-oss` for OSS repo security research with Docker sandboxing, ASAN/libFuzzer/Valgrind harness support, static scanning, per-path reachability triage, and a local session dashboard.
- Added new MCP tools: `bounty_repo_docker_run`, `bounty_repo_check`, `bounty_repo_inventory`, `bounty_repo_prepare_env`, `bounty_import_static_artifact`, `bounty_static_scan`.
- Added Kimi CLI adapter with `hacker-bob-kimi` wrapper package; centralized wrapper registry in `scripts/lib/package-policy.js`.
- Wired CVE feed matching into `bounty_public_intel`.
- Hardened installer writes against symlinks.
- Added PSL staleness warning when the installed Public Suffix List is more than 90 days old.
- Isolated MCP egress profile tests to prevent workspace state interference during `npm test`.
- Extracted implicit runtime behavior into explicit contracts and stores for findings, session state, verification rounds, wave handoffs, grade verdicts, pipeline events, pipeline session artifacts, handoff signing, and session authority.
- Fixed null-vs-missing validation in `mcp/lib/tool-validation.js` for required + nullable fields.
- Added per-session `handoff_provenance_required` flag; v1.3.5+ sessions reject the `legacy_unverified` handoff downgrade path by default.
- Unified wave readiness validation in `bounty_apply_wave_merge`.
- Added deterministic package/release checks: authority inventory freshness, clean release audit, dependency freshness gates in CI.
- Fixed installed policy-replay SDK resolution for both bare specifier and `createRequire`-based fallback paths.

## Review-cycle hardening

The v1.3.5 review cycle surfaced four P1/HIGH blockers and a MEDIUM regression. All are fixed:

- **Null at dispatch.** `validateObject` now accepts explicit `null` for required + nullable fields.
- **Session backfill.** `normalizeSessionStateDocument` runs before `assertLegacyFailClosedFields`; v1.3.4 sessions resume cleanly.
- **Handoff provenance flag.** New `handoff_provenance_required` per-session flag; v1.3.5+ sessions reject `legacy_unverified` downgrade with a clear re-init hint.
- **Unified wave readiness.** `buildWaveReadiness` validates content and provenance inline; `bounty_apply_wave_merge` surfaces invalid agents.
- **Chain-notes resilience.** ENOENT on assignment file falls back gracefully; validation failures still strict-drop.
- **Deprecation shim.** `mcp/lib/pipeline-analytics.js` re-exports four helpers moved to `pipeline-events.js`.

## Known follow-ups (v1.3.6)

- Complete removal of the `legacy_unverified` handoff path (kept behind flag in v1.3.5 for in-flight session drain).
- End-to-end live policy-replay smoke test exercising `query()` invocation.

## Upgrade

```sh
npx -y hacker-bob@1.3.5 install /path/to/your/project
npx -y hacker-bob-cc@1.3.5 install /path/to/your/project
npx -y hacker-bob-codex@1.3.5 install /path/to/your/project
npx -y hacker-bob-kimi@1.3.5 install /path/to/your/project
```

### Pre-v1.3.5 sessions

Sessions started on v1.3.4 and resumed on v1.3.5 inherit `handoff_provenance_required: false` so they can drain to completion. To take advantage of strict provenance rejection on an in-flight session, re-init with `bob init <target>`. v1.3.6 will require this for every session.
