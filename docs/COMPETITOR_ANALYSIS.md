## TL;DR (highest-signal takeaways)

1. **None of the three is a direct competitor to Bob's center of gravity.** raptor owns source/binary/SCA but its live-web side is reportedly a stub; CyberStrikeAI is a red-team console that reportedly makes verification and scope-enforcement *non-goals*; poc-lab is a knowledge corpus with no agent at all. Bob is alone in doing live-web bounty + 6-chain SC audit + OSS source audit through one verified, scope-enforced pipeline.

2. **Bob's #1 acknowledged limit is a dropped producer, not a missing feature.** The OSS reachability/severity-ceiling classifier exists as code at blob `d1647c0` (with `1a456f1`, `9433537`), but it is **not in Bob's working tree on `main`**: `mcp/lib/reachability.js` does not exist and `grep -ci reachab mcp/lib/repo-target.js` = 0. The **entire consumer half is still wired and orphaned** — `assignment-brief.js` (5 refs, with the comment "Stamped … by repo-target.js"), `wave-brief-derivation.js` (4 refs), `dashboard.js` (4 refs), and the evaluator prompt (`prompts/roles/evaluator.md:65-66`) instructs agents on `severity_ceiling`/`attack_vector`/`network_reachable`/`network_reachable_anchors[]`/`residual_hunt_targets[]` fields that nothing populates. The guard tests (`test/repo-target.test.js`, `test/dashboard.test.js`) are absent from `test/mcp-test-manifest.json` and `require` renamed modules, so the regression went uncaught. **The fix is "restore your own code," not "adopt raptor."** Highest value-to-effort move in the backlog. (Precise framing: see gap #1 — there is *no removal commit*; the producer was superseded at a merge, not deleted from a live `main`.)

3. **The skeptic pass demoted or killed roughly half the candidate list** — 6 candidates are *already-covered* (record-time provenance, screenshot hygiene, in-wave steering, model-routing table, exploitation-completeness label, dependency vuln-DB) and several "high value" claims were overstated. It was not perfect, though: it *itself* propagated one error (it listed "reachability demotion" as a live pre-LLM triage input — it is not; see TL;DR #5 and the inconsistency note in the gaps section). Net, believing the *raw* candidate list would still have caused real wasted work.

4. **Several proposed "improvements" are actively dangerous for Bob's mission.** SMT-unsat-skip and "dependency-not-in-CVE-DB-skip" both invert Bob's enforced no-premature-termination doctrine and would suppress the novel 0-days Bob exists to find. A cheap-tier FP prefilter *before* the brutalist verifier inverts freeze-then-verify. Adopt the *signals*, never the *auto-kill gates*.

5. ** raptor's real lesson is economic, not feature-level: it reportedly spends serious effort NOT calling the LLM** (free deterministic refutation, SMT pruning, dead-code suppression, Wilson-bounded prefilter). Bob feeds essentially everything to LLM evaluators and trusts LLM judgment for grade *quality* (only the arithmetic is server-checked). Bob already has *some* pre-LLM triage — `lead-scoring.js` (`shouldPromoteLead`, default `min_score:60`, `limit:8`) and per-surface context budgets — though less than the candidate claimed, and notably **NOT** reachability-based (the stamp is absent, and even when present it never feeds ranking/budgeting; see the inconsistency note). What Bob lacks is *measuring and surfacing* the savings/reliability.

6. **poc-lab's real lesson is disclosure craft.** Bob runs an active GHSA/CVE campaign and the operator hand-transcribes `report.md` into advisory forms, yet Bob's OSS report branch omits even a CWE line (`prompts/roles/reporter.md:28-31` — the HTTP branch at `:38` and the SC branch at `:49` both have CWE). Cheap prompt-only wins live here.

7. **Bob's genuine, novel gaps after the skeptic pass are narrower and cheaper than the architecture docs imply:** restore reachability stamping (and add a consumer that *acts* on it); add a patched-vs-unpatched differential oracle (kills the recurring "this was already fixed" maintainer pushback on residual findings); quarantine untrusted text entering briefs; enrich the OSS report template; broaden root-cause vocabulary beyond memory-safety (two genuinely-absent families, not four); source fuzz seeds. Everything else is park/defer.

## Honest verdict per project

> **Note.** "What it is" / "What X does better" content below was read from the real clones and the load-bearing facts were re-verified (see editor's correction above). "What Bob does better" halves are Bob-side and were checked against the live tree.

### raptor vs Bob

**What it is :** A heavyweight Claude-Code-native source/binary/supply-chain analysis engine (reported "~300K non-test Python LOC / ~916 `test_*.py`" — implausibly large; likely counts vendored/generated code) chaining Semgrep+CodeQL+Coccinelle, AFL++/libFuzzer, Z3/SMT, DWARF+nm reachability, and SCA→SBOM/VEX, with a deliberately deterministic, scanner-driven macro pipeline.

**Peer/threat/inspiration:** A **serious peer in a different room** and Bob's richest inspiration source. Reportedly not a market threat — its `/web` is described as an alpha stub (empty SSRF rules, uninitialized offsec submodule) and it has zero smart-contract capability, so the only true head-to-head surface is OSS source auditing, where raptor is said to be materially deeper. 

**What raptor does better :** Pre-LLM deterministic refutation economics (free gates + savings counters); SMT/Z3 path-feasibility with real C semantics; binary-oracle dead-code suppression with held-out generalization numbers; statistically-honest self-calibration (Wilson bounds, per-`(model,decision_class)` trust cells); agent-self-security (prompt-injection nonce envelope, x-source provenance, sandbox); headless CI mode; CWE-strategy YAML playbooks with a scoring picker; self-growing verified-outcome corpus.

**What Bob does better (Bob-side, verified):** Genuinely exploratory, strategy-forming orchestration; MCP-owned content-addressed event-sourced state authority; live-web + 6-chain SC engines; live-target scope/SSRF/PII safety; 4-host portability with `--check` drift gates.

### CyberStrikeAI vs Bob

**What it is :** A self-hosted Go monolith red-team console — reported ~89 CLI-tool recipes + built-in C2 + webshell manager + host PTY behind an Eino-ADK agent loop, driven conversationally through a vanilla-JS SPA. Operator-trust + access-control posture; finding-correctness and scope-enforcement are described as explicit non-goals.

**Peer/threat/inspiration:** A **peer in runtime/UX engineering, a different category in mission.** Reportedly what Bob would be if you deleted the correctness layer and bolted on a C2. Strong inspiration for observability/UX; near-zero inspiration for correctness.

**What CSAI does better :** Live reconnectable in-run observability (`TaskEventBus` SSE timeline, refresh/multi-tab re-attach mid-run); in-loop HITL steering at tool granularity; runtime context-compaction middleware for long single-agent loops; full kill-chain breadth incl. post-exploitation; swappable orchestration topologies + checkpoint/resume; generic third-party MCP federation; broad integration surface (Burp/DingTalk/Lark/WeChat).

**What Bob does better (Bob-side, verified):** Essentially everything correctness- and safety-related — freeze→3-round adversarial verify→code-adjudicated→5-axis grade vs (reported) pure-CRUD `record_vulnerability`; scope enforcement + SSRF hardening vs (reported) zero target-boundary enforcement; sandboxed runners; registry-driven single-source-of-truth vs (reported) copy-pasted prompt drift; PII guardrails.

**Caution :** CSAI's RAG is reportedly an anti-pattern (sophisticated indexer over a 2-category corpus shipped `knowledge.enabled: false`, brute-force O(N) cosine) — do not copy.

### poc-lab vs Bob

**What it is :** A reported 16-directory, single-squashed-commit human-curated corpus of working exploit PoCs for mostly-2026 high-severity CVEs, each with a deep bilingual root-cause write-up. No agent, orchestration, automation, or machine-readable layer.

**Peer/threat/inspiration:** **Not a peer — a knowledge artifact, and a donor of discipline.** Reportedly ~60% of its content (kernel LPE, browser UAF, binary pwn) is outside Bob's scope; the thin overlap is C-service/parser root-cause clarity (Redis RESTORE, NGINX Rift, Valkey RESP).

**What poc-lab does better :** End-to-end exploitation depth demonstrating HIGH/CRITICAL (vs Bob stopping at "reachable bug + sanitizer repro"); cross-CVE family synthesis; disclosure-grade documentation (CVSS vector, per-distro matrices, fix-commit, timeline, references); reusable known-trigger seed vectors; intellectual honesty as a first-class field (trigger-vs-full-exploit labels).

**What Bob does better (Bob-side, verified):** It is an actual system with verification rigor, safety architecture, breadth, and reproducible machine-readable state. (Note: poc-lab reportedly ships untrusted committed ELF binaries and weaponizes real system files — exactly the "harvest real secrets to confirm" anti-pattern Bob's own rules forbid; but consistent with the read-only handling guidance regardless.)

## Architecture-level lessons

- **Static-analysis layer:** raptor reportedly reveals Bob's OSS engine is mechanically thin — `bob_static_scan` is token-contract-only, no CodeQL/Coccinelle, no SARIF dedup, and no reachability *producer* on `main` (verified). The correct response is *not* to import raptor's binary-oracle wholesale: Bob's `oss_native_code` surfaces are **source checkouts created pre-build**, and Docker builds are best-effort and frequently arch-blocked, so a compiled-artifact oracle is a poor fit at stamp time. Bob's own *source-heuristic* classifier (the dropped producer) is the right architecture; raptor's absence-of-dataflow refutation is an optional precision booster only when a build/CodeQL happens to exist.

- **Fuzzing maturity:** Bob's fuzz path is the least-finished engine (verified) — the `fuzz` recommended-command role exists (`repo-env.js:141`) but is never emitted, seeds come only from the target's own `testdata/`, and aarch64 compiler-rt/libFuzzer friction is documented. raptor's reported host-native AFL++/libFuzzer with reachability-weighted prioritization and plateau detection is the maturity target ; the cheap step toward it is seed-sourcing discipline (Rank 3), not a new harness.

- **Multi-agent patterns:** Bob's "thin, dumb coordinator + MCP-owned event-sourced state + many short-lived stateless agents" is genuinely ahead of all three (Bob-side verified; the comparison rests on notes). Bob should *not* adopt CSAI's reported runtime context-compaction middleware: Bob sidesteps long-context drift by construction (bounded briefs + `resume`). The one place Bob is reportedly behind is **economics** — the "spend effort not calling the LLM" instinct is the philosophy to internalize, expressed in Bob's idiom as *measured* savings/calibration (Ranks 7, 9), never as auto-kill gates that fight freeze-then-verify.

- **Observability/UX:** CSAI exposes Bob's weakest non-correctness axis — there is no live in-flight view (verified: only manual-refresh aggregate + pull tools). This is real but correctly low-priority; the file-backed SSE tail (Rank 8) is the Bob-idiomatic form when the parallel-run itch is felt. Bob should *not* port CSAI's reported in-memory event bus — its `frontier-events.jsonl` ledger already is the event stream.

- **Knowledge representation:** All three reportedly confirm Bob's "knowledge as scored data, not prose-in-prompts" is the right direction but under-populated. The honest constraint: per Bob's own forensic diagnosis, more technique vocabulary improves *coverage*, not the *severity ceiling* — so knowledge enrichment (Rank 5) is corroborating work, not a headline capability. And the enrichment is *smaller* than earlier framing implied — only two families are genuinely absent (crypto-ordering, validate-vs-consume); state-machine confusion is **already** in the pack summary (`technique-packs.js:92`). The structurally interesting move is that Bob already has the "named family + worked witness" format implemented for web3 (`invariant-template-corpus.js` + `TRANSITION_KIND_CONTRACT_TEMPLATES`); extending that shape to OSS is the coherent path, gated behind finishing the unwired `context.ossTechniquePacks` OSS-brief plumbing.

- **Self-defense as a first-class concern:** raptor is the clearest signal that Bob under-invests in treating its *own agents* as an attack surface. Verified Bob-side: Bob redacts secrets in outputs (`sensitive-material.js`, 4000-char cap) but does **not** quarantine attacker-controlled bytes flowing *into* prompts (`audit-report-parser.js` persists them verbatim; 0 datamark/fence hits) — and Bob's own landscape doc names this a CVE-bearing risk. The fix is modest (Rank 6) but the directional lesson is that as Bob ingests more attacker-controlled content (HTTP bodies, audit reports, schemas), prompt-injection hardening should track that growth.
