## 5. Sequencing

- **Δ1 is the right *intent* but its GATE is under-resourced and may strand the whole tier ladder.** Δ1's gate (`TOPOLOGY.md:109`) = "≥1 real OSS finding lifted to a *defensible* HIGH a maintainer accepts." Per §1 above, C9 addresses only 1 of ~6 documented ceiling causes. Critically, **"record-time proof gate downgrades on harness block" overrides a grade-time lift**: if the aarch64/compiler-rt harness is blocked (a recurring reality per `feedback_bob_oss_aarch64_libfuzzer_harness`), the finding is downgraded at *record time* before C9's *grade-time* gate ever runs — and the compiler-rt fix lives in C12 at **Δ3**. So Δ1's gate is coupled to Δ3 work and to uncaptured causes. Since the doctrine is "no tier N+1 without tier-N findings," a Δ1 gate that can't pass **stalls the entire roadmap at Δ1.** Fix: either (a) widen C9 to touch the record-time proof-gate/harness-block path and add chain-severity-elevation into Δ1, or (b) soften the Δ1 gate to "stamp present + consumed + lifted in grade" and move the *maintainer-accepted defensible HIGH* milestone to a Δ1/Δ2 boundary gate after the harness fix lands.

- **Δ1 ride-along IP8-lite advances throughput, not the ceiling.** Fine to ride the OSS code path, but it does not move the Δ1 GATE — don't let it pad the tier's apparent value. C14-lite (CWE+CVSS+refs) is the legitimately high-leverage ride-along (directly cuts disclosure-campaign hand-transcription).

- **Tier moves required:** C14-full → resolve I12 edge (demote to augment) so it stays Δ2 (§3); X8 → Δ4 (§4); C15 → keep Δ4 but scope down or fold into C10 (§4). Everything else's tier is defensible and matches the backlog's "Recommended next 3 moves" ordering (restore+consumer → prompt-only quick wins → differential oracle).

**Net:** the topology faithfully covers the *named* backlog (Ranks 1-9, all architecture lessons, all dangerous-to-adopt rejections).
