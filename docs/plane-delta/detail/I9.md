## Spec (one-paragraph contract)
**Inputs:** the file list + `projection` already computed inside `buildRepoInventory` (`mcp/lib/repo-target.js:775`, `buildInventoryProjection` at `:687-750`), plus bounded reads of native/server source. **Outputs:** (a) an `inventory.reachability` summary object `{ max_credible_severity_ceiling, network_reachable, network_reachable_surface_ids[], surface_ceilings[], signals[] }` attached to the persisted `repo-inventory.json`; (b) per-surface `{ network_reachable, attack_vector, severity_ceiling }` stamps on the `surface.observed` payloads emitted via `emitSurfaceObserved` (`:663-686`). **Predecessor:** S15 (OSS-container substrate, Δ-new). **Unlocks:** C9 (grade-time ceiling lift/cap); augments C11 (static-driven source audit). **Context budget:** producer reads ≤ `NETWORK_SCAN_LIMIT`(50) priority files + ≤ `NETWORK_FALLBACK_LIMIT`(80) native files, each capped at 150 KB, and emits only path-keyed signals (`net_call:<rel>`, `expose:<rel>`, `systemd_unit:<rel>`) capped at 14 — never file contents (O-P7).

## What it does / holds
It **stores** a realistic severity ceiling per source surface, derived from whether the repo contains an in-process network listener that can feed attacker-controlled bytes into a parser. It enables one bounded query a triager recognizes: *"is this AV:N (CRITICAL-capable daemon) or AV:L (MEDIUM-realistic local-only parser)?"* The classifier (re-ported from `d1647c0:mcp/lib/repo-target.js:110-211`) keys off socket/RPC/TLS call signatures (`NETWORK_CONTENT_RE`: `socket|listen|accept4|svc_run|xdr_[a-z]|MHD_start_daemon|SSL_accept…`), network constants (`NETWORK_TOKEN_RE`: `INADDR_ANY|SOCK_STREAM|AF_INET6?…`), systemd `.service`/`.socket` units, server-path heuristics, and Dockerfile `EXPOSE`. `severityCeilingForSurface(oss_native_code, networkReachable)` returns `critical` when reachable else `medium`; supply-chain (`oss_dependency`/`oss_ci_cd`) is pinned `medium` so a MITM lead never masks a memory-safety target's true ceiling. The finding it ultimately produces (through C9): a HIGH/CRITICAL stamp on a network-reachable native parser that a maintainer recognizes as "remotely triggerable," instead of the blind-fuzz MED/LOW plateau diagnosed in `project_bob_oss_severity_ceiling_diagnosis`.

## Failure mode it prevents
The exact regression that already shipped: a network-reachable RCE-capable surface (netatalk afpd, nfs-ganesha NFSACL — the corpus's only real HIGHs) gets routed and graded with no AV signal, so the evaluator pursues a local-only ceiling and the grader caps it MEDIUM. The severity-ceiling diagnosis names "no reachability/severity_ceiling field + no acceptance gate" as the *dominant* root cause of Bob's MED/LOW ceiling. Restoring the producer is the substrate half of fixing it (C9 is the gate half).
