## Spec (one-paragraph contract)
Make the already-shipped OSS technique-pack content reachable in evaluator briefs by wiring the orphaned consumer at `mcp/lib/assignment-brief.js:403` (`briefSliceEntry("technique_packs", 8192, (context) => context.ossTechniquePacks)`), which today has no producer. **Inputs:** an OSS-profile assignment (`brief_profile: "oss"`), the assigned repo surface, the assignment's `task_lens`, and the frozen `OSS_TECHNIQUE_PACKS` registry (`mcp/lib/technique-packs.js:81-124`). **Output:** a bounded `technique_packs` brief slice (`{selected, other_applicable, selection_limits}`) carrying the 7 OSS packs partitioned by `lens_affinity`, plus the sibling OSS slices (`governance`, `goal_orientation`, `code_surface_pack`, `recap_and_handoff`) the `OSS_BRIEF_SLICE_REGISTRY` (`assignment-brief.js:393-411`) declares but no producer fills. **Predecessor:** S5 (SHIPPED — MCP tool registry + role bundles + generators; `capability-hypergraph.md:50`). **Unlocks:** I12 (Δ3 content). **Enforced context budget:** the `technique_packs` slice stays ≤ 8192 chars (the `budget_chars` declared at `assignment-brief.js:403`); the full OSS brief sums the per-slice caps at lines 397-410 (repo_workflow 2048 + governance 1024 + goal_orientation 1024 + code_surface_pack 4096 + technique_packs 8192 + cli_tools 2048 + recap_and_handoff 2048).

## What it does / holds
This is index-wiring, not a finding-producer. It connects an existing index (`OSS_TECHNIQUE_PACKS`, 7 frozen records each carrying `lens_affinity` + hunting-vocabulary `summary`, `technique-packs.js:81-124`) to an existing-but-unreachable consumer (`OSS_BRIEF_SLICE_REGISTRY`, `assignment-brief.js:393-411`). The bounded query it enables: given an OSS evaluator's `task_lens` (one of `code_surface_scout` / `taint_trace` / `fuzz_run`, `assignment-brief.js:361`), return the OSS packs whose `lens_affinity` includes that lens **foregrounded** under `selected`, and the rest **demoted** to `other_applicable` (mirroring the web partition at `assignment-brief.js:292-313`). After this lands, an OSS evaluator that calls `bob_read_assignment_brief` receives `technique_packs.selected` naming e.g. `oss_native_code`'s bounds-check / use-after-free / integer-truncation vocabulary on a `taint_trace`/`fuzz_run` wave — exactly the brief contract the runner skill already advertises at `bob-evaluate-runner/SKILL.md:164` ("the evaluator brief leads with … the OSS technique packs (`oss_dependency`, `oss_native_code`, …)").

## Failure mode it prevents
The shipped state is a documented-but-undelivered contract: two stacked orphans. (1) `assignment-brief.js:403` reads `context.ossTechniquePacks`, which no producer ever sets — and the dispatch `buildBriefExtrasForProfile` (`assignment-brief.js:838-846`) only branches on the web and smart-contract registries and **throws `Unsupported brief profile: ${profile}`** for `"oss"`. (2) The OSS capability packs route *around* the OSS registry entirely: `ossCapabilityPack(...)` hard-codes `brief_profile: "web"` (`capability-packs.js:75`), so OSS evaluators today silently get the **web** brief (HTTP bypass tables, `selectTechniquePacksForSurface` web packs) instead of OSS vocabulary. The bad outcome this stops: an OSS evaluator dispatched on a native-C repo under `fuzz_run` reads a curl-shaped web brief with zero memory-safety hunting vocabulary, while the runner skill and the OSS lens stanza (`REPO_WORKFLOW_TEXT`, `assignment-brief.js:367-391`) promise repo-shaped guidance — a substrate-vs-prompt divergence that quietly degrades every OSS hunt and is invisible because no test exercises `readAssignmentBrief` on an `oss`-profile assignment.

## Risks & doctrine compliance
- **index-without-consumer (the central anti-pattern this node *fixes*):** the work is precisely closing an orphaned consumer (`assignment-brief.js:403`) — but it must not create a *new* orphan. Build slice 7 (stigmergy registration) is mandatory, not optional, or the X9 coherence gate (`check-stigmergy-coherence.js`) would flag the new field-pair.

- **context-as-strategy:** the brief gives vocabulary, not a script; the evaluator still chooses surfaces. The slice is capped (8192) so it informs without crowding out the evaluator's own reasoning budget.
- **auto-kill / freeze-then-verify / no-premature-termination:** N/A and respected — this node touches only brief composition; it adds no lifecycle gate, no `surface_status: complete` shortcut, and cannot suppress a finding.
- **Policy constraints:** Plane-O **O-P1** (`--network none`, init-never-clones) is untouched — this is read-side brief assembly, no container/clone path. The operator-local **hard no-PII rule** is upheld: OSS pack summaries are synthetic, non-personal vocabulary; the producer reads no operator identifiers and no repo secret material into the brief.
