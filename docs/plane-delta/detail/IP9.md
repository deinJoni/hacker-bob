# IP9 — Continuous CVE/commit feed (policy-gated)

**Kind / Tier / Type:** IP · Δ4 (policy-gated) · net-new   **Verdict from verification:** REVISE — not thin. The "thin dispatch wrapper" framing was wrong; IP9 is double-blocked (unbuilt IP3 watcher + an absent temporal scheduler substrate), it resurrects the deliberately-vacated C8, and a cron loop with no human between fetch and dispatch strains Bob's operator-gated, two-turn interaction model. Keep Δ4/policy-gated; build the two prerequisites as explicit sub-work, not as a wrapper.

## Spec (one-paragraph contract)
**Inputs:** on each scheduled tick, a fresh operator-supplied CVE feed (`cve_feed_json` text, exactly as `bob_public_intel` already takes it — `mcp/lib/public-intel.js:381`) and/or a fresh commit/diff set for an authorized, locally-checked-out repo; the prior per-`(target, feed_source)` delta cursor; and a per-program `ai_disclosure_policy` authorization flag. **Outputs:** the *delta* set of newly-matched, in-scope CVE rows (via I8) and newly-changed watched surfaces (via IP3), each content-addressed; and — only behind the policy gate — a record of dispatched waves/repo-sessions. **Predecessors:** IP5 (`cve-feed-parser.js`, SHIPPED), I8 (`cve-scope-matcher.js`, SHIPPED but consumed only pull-mode), IP3 watcher (UNBUILT), temporal scheduler substrate (ABSENT). **Enforced context budget:** reuse the public-intel caps — `PUBLIC_INTEL_MAX_RESPONSE_BYTES = 300_000`, `PUBLIC_INTEL_MAX_ITEMS = 10` (`mcp/lib/constants.js:91-92`); the per-tick delta summary handed to the orchestrator never exceeds those.

## What it does / holds
The genuine net-new artifact IP9 owns is a **content-addressed delta cursor**: per `(target_domain, feed_source)`, the set of `cve_record_hash` (`cve-feed-parser.js:189`) / `match_hash` (`cve-scope-matcher.js:168`) and watched-repo commit SHAs that have already been seen and (if policy-permitting) dispatched. The bounded query it enables: "given this newer feed, what is *new since the last tick* that matches an authorized target's scope?" Everything downstream already exists — `parseCveFeed` (`cve-feed-parser.js:193`) → `matchCveBatch` (`cve-scope-matcher.js:172`) → scope-filtered rows is exactly what `buildCveScopeMatches` does today (`public-intel.js:380-444`), keyed for freshness to `attackSurfaceFingerprint` (`public-intel.js:369`). IP9 adds only the *temporal delta* and the *scheduled re-invocation*; it does not re-implement parsing or matching. As an ingestion path it is not parishioner-visible on its own — it is useful only when the index/dispatch it feeds produces a finding (per the IP review rule, `capability-hypergraph.md:282`).

## Failure mode it prevents
Two concrete bad outcomes. (1) **Stale disclosure posture:** a freshly-published CVE lands on an authorized target's `tech_stack`, or a watched repo regresses a fixed bug, and Bob never notices until the next *manual* `/bob-evaluate` — the disclosure campaign goes blind between human runs. (2) **Duplicate-dispatch stampede:** naive periodic re-ingest of the same feed re-matches and re-dispatches the *same* CVEs every tick (runaway waves, runaway cost). The content-addressed cursor makes a tick emit only the delta; the same feed twice → empty dispatch set. Note: IP9 must NOT prevent "unknown" CVEs from being acted on — the feed is a dispatch *signal*, never a kill gate (see Risks: X-rej-1).

## Predecessors & unlocks
- **IP5 — `cve-feed-parser.js` (SHIPPED).** `parseCveFeed`/`parseNvdEntry`, content-addressed `cve_record_hash`. Reused as-is.
- **I8 — `cve-scope-matcher.js` (SHIPPED, but consumed only pull-mode in `public-intel.js`).** `matchCveAgainstScope`/`matchCveBatch`, `match_hash`, `cve_version_range`. Reused as-is.
- **IP3 watcher (Δ-new, UNBUILT).** Only the diff *parser* ships (`mcp/lib/unified-diff-parser.js`); the repo-poller/watcher does not exist — `grep repo.?watch` is empty and the C3 status admits "an actual repo-poller (cron-style watcher) is operator-driven for first slice" (`capability-hypergraph.md:319,322`). **Cross-node blocker.**
- **Temporal scheduler substrate (Δ-new, ABSENT).** Bob's `graph-scheduler.js` (`GRAPH_SCHEDULED_KINDS = ["transition","hypothesis"]`, `selectNextExecutableNodes`) and the wave-scheduler are **within-session** dispatchers; there is no cross-time/cron substrate and no `bob_*cron*`/`*feed*`/`*watch*` MCP tool (`ls mcp/lib/tools`). **Substrate blocker.**
- **Unlocks:** temporal C13 (cross-target transfer on new CVEs), **C17** (proof-carrying regression sentinel — `nodes.json` C17 lists IP9 as a hard predecessor), and resurrects **C8** (live disclosure speedrun, deliberately vacated `capability-hypergraph.md:386,521-529`). **Cross-tier:** all three consumers are Δ4; C17 additionally needs C14 (Δ2) and S1.

## DO (build slices)
Honest ordering: IP9's own deliverable (delta cursor + scheduled re-invocation) is small; its two prerequisites are real builds tracked as separate nodes. Slices below scope IP9 itself and name the prerequisite hand-offs explicitly rather than pretending they are free.

1. **Spec / decide (one commit, doc-only).** Write `docs/plane-delta/detail/IP9.md` contract: the temporal model, the policy gate, and the hard statement that IP9 is *parked until* (a) IP3 watcher ships, (b) a temporal scheduler substrate ships, (c) ≥1 program grants explicit AI-disclosure policy (the same gate that holds C8, `capability-hypergraph.md:527`). No code here — this slice exists so the node is not silently half-built.

2. **Delta-cursor substrate (net-new, IP9-owned).** New file `mcp/lib/temporal-feed-state.js`: a per-`(target_domain, feed_source)` JSONL/store of already-seen `{cve_record_hash, match_hash}` and commit SHAs, content-addressed and idempotent on re-write (mirror the `hashCanonicalJson` + upsert pattern in `cve-scope-matcher.js:168` and `audit-reports.jsonl` ingest). Export `diffFeedAgainstCursor({ target_domain, feed_source, parsed_records, matched_rows, prior_cursor })` → `{ new_rows, advanced_cursor }`. **No network, no dispatch in this slice.**

3. **Delta MCP tool (net-new, ingest-only).** New `mcp/lib/tools/ingest-temporal-feed.js` registered through the registry (`mcp/lib/tools.js` + role bundles via the generators — `node scripts/generate-agent-tools.js`). It calls the *existing* `buildCveScopeMatches` (`public-intel.js:380`) then `diffFeedAgainstCursor` (slice 2) and returns only the delta, capped at `PUBLIC_INTEL_MAX_ITEMS`. This tool is **ingest/read-shaped** — it advances the cursor but does **not** dispatch. Feed text stays operator-supplied: `assertAllowedPublicIntelUrl` allowlists only `hackerone.com` (`public-intel.js:134-140`), so Bob cannot itself fetch an NVD feed — that constraint is inherited, not relaxed.

4. **Prerequisite hand-off A — IP3 watcher (separate node; gating).** IP9's commit half consumes a repo-watcher that produces unified diffs for an authorized, *locally-checked-out* repo (O-P1: init never clones — `mcp/lib/tools/init-repo-session.js:30,38`; `--network none` — `mcp/lib/repo-env.js:407,838`). Wire point when it lands: feed its diffs through `parseUnifiedDiff` (`unified-diff-parser.js`) → `summarizeImpactedSurfacesForDiff` → cursor. **Do not stub a network clone to fake this.**

5. **Prerequisite hand-off B — temporal scheduler substrate (separate node; gating).** The cron-tick itself is external to Bob's MCP today. The honest minimal form: a Claude-Code routine / harness cron invokes `bob_ingest_temporal_feed` on a cadence; Bob's MCP stays the durable delta-cursor store, not the timer. If/when an MCP-owned temporal substrate is justified, it is its own slab node — IP9 must not smuggle a `setInterval` into the MCP server.

6. **Policy-gated dispatch (LAST; behind the C8 gate).** Only after slices 2-5 *and* explicit program policy: a thin orchestrator routine reads the delta (slice 3), and for each new in-scope row dispatches the existing wave system (`bob_init_session(targetB)` + `bob_start_wave`, single-domain today) or `bob_init_repo_session` for the commit half — exactly the wrapper C8 describes (`capability-hypergraph.md:529`). The per-program `ai_disclosure_policy` check is enforced **in the dispatch routine, not in a prompt** (same posture as C13's scope-auth gate), and the cursor advances atomically with the dispatch record (slab S2).

- **Context-budget compliance:** per-tick delta ≤ `PUBLIC_INTEL_MAX_ITEMS = 10`, feed text ≤ `PUBLIC_INTEL_MAX_RESPONSE_BYTES = 300_000` (`constants.js:91-92`); assert the cap is enforced, mirroring `public-intel.js:382-383`.
- **Determinism / content-hashing:** same `(feed, prior_cursor)` → byte-identical `new_rows` and `advanced_cursor`; keys are `cve_record_hash`/`match_hash`/commit SHA (no wall-clock in the canonical payload — the determinism-CI lesson at `capability-hypergraph.md:418`).
- **Failure-mode coverage:** malformed feed (`parseCveFeed` warnings path), partial tick (dispatch started, cursor not advanced → must roll back, never half-state), orphan cursor (`(target,feed_source)` whose target was deleted), and the stampede case (cursor missing → must refuse to dispatch, not re-fire everything).
- **Slab invariants to preserve (`capability-hypergraph.md:57`):** **S1** content-addressing (cursor keyed on hashes); **S2** manifest-coupled atomic advance (cursor + dispatch record commit together or not at all — this is the anti-stampede invariant); **S3** replay-lease typing (a temporal loop firing waves must honor per-pack `serialized`/`parallel_safe` leases and rate limits so it cannot stampede); **S7** credentials never enter context (feed ingest is CVE/commit text only).

## REVIEW (parishioner / real-target gate)
**Long-horizon and policy-blocked — state it plainly.** The honest engineering-only gate that closes IP9's *own* deliverable: on a seeded two-tick fixture, tick 1 ingests a feed and records N matched in-scope rows; tick 2 with one added CVE returns exactly that one new row and the cursor advances by one — no re-emission of the N. The real-target/disclosure gate (a newly-published CVE auto-routed to a wave that yields a triager-accepted finding) **cannot be closed** until (a) IP3 watcher and the temporal substrate ship, and (b) a program grants explicit AI-disclosure policy — the same external dependency that holds C8 (`capability-hypergraph.md:521-529`). Until then IP9 stays parked behind its fixture gate; do not present the fixture pass as a real-target win.

## Context budget
- Per-tick delta summary to orchestrator: ≤ `PUBLIC_INTEL_MAX_ITEMS = 10` rows; feed text ≤ 300 KB (`constants.js:91-92`).
- No new long-lived subagent brief — dispatch (slice 6) reuses existing wave briefs at their existing caps; IP9 adds no extra brief slice, so it introduces **zero** new subagent context budget.
- Cursor store excerpt read into any brief: cap to the same 10-item ceiling; never inline the full historical cursor.

## Risks & doctrine compliance
- **Auto-kill that inverts no-premature-termination (X-rej-1):** the cursor must gate *re-dispatch of already-seen items*, never *suppress* a CVE for being "unknown / not in DB." The feed is a dispatch signal; an unmatched/novel CVE is still huntable via normal flow. Encode "delta gate, not kill gate."
- **Substrate-as-trajectory:** building the temporal scheduler substrate is **not** itself parishioner progress (`capability-hypergraph.md:32`). The two prerequisite builds advance no tier on their own — do not let "we built the scheduler" count as an IP9 finding.
- **Index-without-consumer:** the cursor is justified only because C17/C13-temporal/C8 consume it. If those stay unbuilt, IP9 stays parked (slice 1) — don't ship a cursor with no consumer.
- **Context-as-strategy:** delta is bounded by hash-diff, not by stuffing the whole feed history into a brief.
- **Freeze-then-verify:** dispatch (slice 6) feeds the *normal* hunt→freeze→verify pipeline; IP9 must not pre-judge or pre-close anything it dispatches.
- **Plane-O O-P1 (`--network none`, init-never-clones):** the commit half consumes operator-checked-out repos only (`tools/init-repo-session.js:30,38`; `repo-env.js:407,838`); IP9 introduces no in-container or MCP-side clone/fetch. The CVE feed remains operator-supplied (allowlist is hackerone-only, `public-intel.js:134-140`).
- **Hard-rule no-PII:** feed/commit text carries no operator identifiers; nothing from operator context is submitted to any target — dispatch only re-runs the existing scoped pipeline.
- **Frontier safety/scope gate:** because IP9 unlocks autonomous, scheduled action against authorized targets, the per-program `ai_disclosure_policy` authorization is a **hard gate enforced in the dispatch routine** (not a prompt), and IP9's dispatch half stays Δ4/policy-gated until that policy exists — preserving the operator-gated, human-in-the-loop posture this node would otherwise erode.
