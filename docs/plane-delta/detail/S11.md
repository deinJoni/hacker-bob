# S11 — Cross-session label/metric store

**Kind / Tier / Type:** S (slab) / Δ4 / adopt   **Verdict from verification:** revise → re-typed net-new→**adopt** (WEAVE.md:7). The cross-session read substrate already exists; S11 extends it. Drop the planned `~/.cross` dir; drop the stale "parked I6 prereq" rationale (I6/`findings-index.js` was deleted in `c02179c`, so labels must source from grade verdicts + adjudication, not the removed `calibration_label` slot).

## Spec (one-paragraph contract)
**Inputs:** at grade time, the frozen verdict (`writeGradeVerdict`, `grade-verdict-store.js:311`), the final reportable-severity set already computed there (`:350`), the adjudication disposition summary (`verification-round-store.js:392`, `confirmed`/`denied`/`downgraded` per `constants.js:58`), plus the run counts (`findings`, `total_score`) already attached to the `grade_written` event (`:364-373`). **Outputs:** (a) a per-session, append-only, content-addressed `label-metrics.jsonl` of sanitized **label records** written under the existing session root (`paths.js:37-42`, `~/bounty-agent-sessions/<domain>/` — **no** `~/.cross`); (b) a bounded cross-session **aggregate projection** surfaced inside `readPipelineAnalytics` `mode:"cross_session"` (`pipeline-analytics.js:1150-1169`). **Predecessors:** S1, S2 (both SHIPPED). **Enforced context budget:** per-session record ≤300 tokens; cross-session labels block ≤3k tokens; session fan-out capped at `CROSS_SESSION_ANALYTICS_MAX_SESSIONS = 200` (`pipeline-analytics.js:41`). Every write and every read passes `validateNoSensitiveMaterial` (`sensitive-material.js:73`, 4000-char cap at `:7`); the cross-session read inherits the existing `cross_session_read` authority class (`session-authority.js:356-387`) with `index_only_no_target_url_export`.

## What it does / holds
It **stores** one sanitized label record per graded session: `{ session_key (hashed domain), model, attack_class, decision_class, verdict ∈ {SUBMIT,HOLD,SKIP} (constants.js:73), disposition_counts ∈ {confirmed,denied,downgraded}, severity_histogram, findings_count, total_score, label_hash }`. `decision_class` is the joined grade×adjudication cell I11 needs. Records are append-only and content-addressed (`label_hash = hashCanonicalJson(record − timestamps)`), so re-grading the same frozen batch upserts on the same hash. The **bounded query it enables**: "across the last N sessions, give me per-`(model, attack_class, decision_class)` outcome counts" — the exact cells I11's Wilson lower-bound and X7b's per-class economics consume, returned as a small aggregate, never raw findings or response bodies. There is no per-finding text and no `target_url` in the store; it is an index of derived labels, not a copy of evidence.

## Failure mode it prevents
Two concrete bad outcomes. (1) **Outcome amnesia across sessions** — without a persisted, queryable label store, each evaluate restarts with zero memory of how a `(model, attack_class)` cell actually graded out historically, so I11's calibration prior has nothing to read and the operator can never answer "is opus over-claiming reentrancy on web targets?" The existing cross-session reader (`pipeline-analytics.js`) aggregates only operational health (tool funnel, bottlenecks), not graded outcomes. (2) **PII/secret leakage through a "convenience" cross-session cache** — the naive fix (dump findings into a shared `~/.cross` dir) would violate the hard no-PII rule and the `index_only_no_target_url_export` authority policy. S11 prevents that specific regression by storing only sanitized derived labels, validated on write and on read.

## Predecessors & unlocks
- **S1 — Content-addressed evidence (SHIPPED).** Provides `hashCanonicalJson` / `final_verification_hash` semantics; S11 keys each label record on a content-addressed `label_hash` so re-grades dedupe and replay is byte-stable.
- **S2 — Manifest-coupled state (SHIPPED).** Provides atomic-transition / session-lock guarantees; S11 writes the label record inside the same `withSessionLock` block as `writeGradeVerdict` (`grade-verdict-store.js:313`) so a grade can never land without its label (no half-state).
- **Unlocks I11 (Δ4, Δ-new)** — calibration ledger / trust cells; I11's anchor is literally "persist labels on `bob_write_grade_verdict` → S11 store" + a Wilson-bound read. **Unlocks X7b (Δ4, Δ-new)** — cost-per-finding economics (the heavy half split from X7; X7 itself needs neither S11 nor I11).
- **Cross-tier dependency flag:** S11 is a Δ4 slab whose only consumers (I11, X7b) are also Δ4 and gated **engineering/fixture, long-horizon** (calibration is "dormant at operator volume", WEAVE.md:53). Do not schedule S11 ahead of I11 as standalone substrate — see index-without-consumer risk below.

## Risks & doctrine compliance
- **substrate-as-trajectory:** S11 is slab but has named consumers (I11, X7b); ship it paired with at least I11's read path so it is not dead substrate. Self-check passes only if a consumer reads `labels` in the same PR series.
- **index-without-consumer:** until I11 lands, labels are written-but-unread. Acceptable *only because* S1/S2 make them replayable and the write is near-zero-cost (rides the existing grade lock + event append). Flag in the PR that the cross-session `labels` block is consumer-pending.
- **context-as-strategy:** mitigated by hard caps (300/3k/200), index-only projection, and zero raw bodies — the store cannot become a "stuff everything in context" cache.
- **auto-kill / freeze-then-verify inversion:** labels are **descriptive and non-binding** (I11 is explicitly a non-binding prior). S11 must never feed a HUNT→kill, a grade-time auto-suppress, or any premature-termination gate. No write path may consult the store before recording a candidate or verdict.
- **Policy constraints:** hard-rule **no-PII** — store only derived labels (verdict, disposition, severity histogram, counts, model, attack_class); never operator email/identifiers, finding prose, response bodies, or `target_url`. `cross_session_read` authority enforces `index_only_no_target_url_export` (`session-authority.js:384`) — the aggregate keys on a hashed `session_key`, never the raw domain. Plane-O `--network none` is not engaged (no container/clone path). No frontier safety/scope gate beyond the existing cross-session authority class.
