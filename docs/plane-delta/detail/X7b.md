# X7b — Cost-per-finding telemetry
**Kind / Tier / Type:** X (cross-cutting) / Δ4 / net-new   **Verdict from verification:** split (the heavy `$`-cost half carved off X7; X7 keeps the Δ2 `evaluator_run_avoided` ride, X7b owns the token-telemetry substrate Bob lacks — WEAVE.md roll-up row X7, nodes.json `split_child:"X7b"`).

## Spec (one-paragraph contract)
**Inputs:** (a) the Claude Code per-turn transcripts already written to disk for a run — root session jsonl + `subagents/agent-<id>.jsonl` + `.meta.json` (each turn carrying `message.usage` = `{input_tokens, output_tokens, cache_creation_input_tokens, cache_read_input_tokens}` and a per-turn `model` ID; per `docs/BOB_OSS_BENCHMARK_PLAN.md` §7.3, *assumed* — this is Claude Code harness state, not a Bob-owned artifact, so the on-disk schema is not greppable in `mcp/lib`); (b) a versioned per-model price table; (c) the session's finding count from `verified-final.json`/`grade.json`; (d) X7's `evaluator_run_avoided` count for the calls-avoided half. **Outputs:** a per-session economic record `{session_id, target_domain, price_table_version, tokens_by_model_and_class, usd_total, findings_confirmed, usd_per_finding, calls_avoided, usd_saved, reconciliation_status, cost_record_hash}`, persisted into S11's cross-session metric store and surfaced by a new orchestrator-only `bob_read_cost_metrics` read tool + a `bob-status` panel row. **Predecessors:** S11 (Δ-new) for cross-session persistence; soft cross-tier dep on X7 (Δ2) for `calls_avoided`. **Unlocks:** economic-accountability. **Enforced context budget:** cost slice surfaced to orchestrator ≤2k tokens; per-session economic row ≤300 tokens; the reconstruction itself runs in a deterministic module (no LLM in the loop, no raw transcript ever enters a brief).

## What it does / holds
It is the index Bob has no native producer for. Verified: Bob's tool telemetry event (`mcp/lib/tool-telemetry.js:386-405`) records only `version, bob_version, ts, tool, ok, elapsed_ms, error_code, target_domain, wave, agent, surface_id` — **no token counts, no `model`, no USD**; `PIPELINE_EVENT_TYPES` (`mcp/lib/pipeline-events.js:28-52`, 23 frozen types) likewise carries no cost type. So **timing is fully derivable today** (`pipeline-events.jsonl` ISO `ts` + `elapsed_ms` p50/p95 at `mcp/lib/tool-telemetry.js:844-846`; benchmark plan §7.2) but **dollars are not** (§7.3, "confirmed by source grep").

X7b stores: per-(session, `agentType`, `model`, token-class) aggregates reconstructed from the transcripts; the priced USD total under a **pinned `price_table_version`** (mandatory per-turn per-model pricing — one real run mixes `claude-opus-4-8`→`claude-sonnet-4-6` and cache-read dominates volume at a far lower price than fresh input, so a single blended rate is wrong; `<synthetic>`-model turns carry no API cost and are excluded — §7.3); and the derived economics: `usd_per_finding`, and `usd_saved = calls_avoided × per-call-cost`.

The "finding a triager recognizes" framing does not apply — this is an observability node, not a capability that produces vulnerability findings. The bounded query it enables: "what did the netatalk run cost per confirmed advisory, and how much did lead-filtering save?" answered from a ≤300-token row, not by stuffing transcripts into context.

## Failure mode it prevents
Two. (1) The operator-facing one: a disclosure campaign (netatalk/openexr/libde265…) with **no denominator for its own economics** — "is `/bob-oss` worth running on target X?" has no answer because tokens/USD were never recorded; every cost claim is hand-waved. (2) The benchmark-integrity one: `docs/BOB_OSS_BENCHMARK_PLAN.md` makes "$ + tokens + wall-clock per confirmed distinct bug" a first-class Phase-1 result (§8) but flags it as the **primary instrumentation gap** (§7.3); without this node every dollar figure stays "(derived externally — reconciliation pending)" forever and the benchmark falls back to wall-clock-only.

(assumed) The transcript path/field layout follows benchmark plan §7.3; gate it behind a tolerant parser that emits `reconciliation_status:"no_transcripts"` and falls back to wall-clock if the files/fields are absent — never throw, never invent a number (mirror the "malformed input → warning+skip" rule the schema/audit parsers already follow).

Any consumer rendering a `derived-pending` row MUST label `usd_per_finding` "(derived externally — reconciliation pending)" and show wall-clock as the fallback cost axis (§7.3 framing discipline). The independent count is operator-side/offline — **not** fetched in-session.

## REVIEW (engineering)
- **Tests** — new `test/cost-telemetry.test.js`: per-turn-per-model pricing (opus+sonnet mixed run prices correctly; `<synthetic>` excluded); cache-read priced below input; `cost_record_hash` reproduces byte-stably across two runs of the same fixture transcript and **diverges on `price_table_version` change** (so a silent rate edit can't masquerade as the same number); missing-transcript fixture → `"no_transcripts"` + wall-clock fallback, no throw; reconciliation thresholding at the ±5% boundary; sanitizer drops a planted secret/PII field before persistence. Register the module→guard-test pair in S12's manifest coverage check so this producer can't be silently dropped the way I9's was.
- **Context-budget compliance** — surfaced cross-session slice **≤2k tokens**; per-session economic row **≤300 tokens**; reconstruction runs in-module with **no LLM subagent** (raw transcripts never enter context). If a cost-anomaly triage subagent is ever added, ≤3k.
- **Determinism / content-hashing** — `cost_record_hash` content-addressed; `$` figures reproducible **only against a pinned `price_table_version`** (prices are external + time-varying — state this in the record, don't pretend the dollar number is intrinsically deterministic).
- **Failure-mode coverage** — orphan (subagent jsonl present, `.meta.json` missing → attribute to `unknown` role, don't drop tokens); partial (root present, backgrounds missing → `"partial"` status, never silently undercount); malformed (non-JSON turn → skip+warn).
- **Slab invariants preserved** (`docs/capability-hypergraph.md:57`): no credential/PII leakage into the cross-session store (S11 + sensitive-material); artifacts content-addressed; **no lifecycle advance** — cost telemetry is read-only observability and must never touch `pending_wave`/`evaluation_wave`/manifest state (consistent with the evaluator-lifecycle-hook rule in CLAUDE.md); cross_session reads respect `cross_session_read` authority typing.

## REVIEW (parishioner / real-target gate)
**Honest call: this is an ENGINEERING/FIXTURE gate, long-horizon — not a real-target finding gate** (WEAVE.md must-fix #11: X7/I11 "can't pass at operator volume"). It produces no vulnerability finding, so the `docs/capability-hypergraph.md:512-517` parishioner ladder (count findings / FPs / triager-recognition) does not apply. The closing criterion is the benchmark §8 Phase-0 **cost-join reconciliation gate**: on ≥3 real `/bob-oss` sessions (root + background evaluators), the reconstructed per-session token total reconciles **within ≈5%** of an independent Anthropic usage count; once it does, `reconciliation_status` flips to `"reconciled"` and a real multi-finding campaign session (e.g. a netatalk-class run with several advisories) yields a defensible `$/finding`. **Dormancy caveat:** at operator volume (often 1-few findings/session) `$/finding` is descriptive, not statistically meaningful — surface it, never headline it.

## Context budget
- Cross-session cost slice surfaced to the orchestrator / `bob_read_cost_metrics` response: **≤2k tokens**.
- Per-session economic row (the unit S11 persists and `bob-status` renders): **≤300 tokens** (matches the I4 per-row cap, `docs/capability-hypergraph.md:166`).
- Reconstruction module: **no subagent, no brief slice** — deterministic file parse. Any future cost-anomaly triage subagent: **≤3k tokens**, and it receives the aggregated row only, never raw transcript text.

## Risks & doctrine compliance
- **Substrate-as-trajectory (anti-pattern #1 / #4 index-without-consumer / #5 cathedral cadence, `docs/capability-hypergraph.md:533-537`):** the live risk — a token-telemetry substrate with no consumer. **Mitigation: never land slices 1-4 without slice 5** (the `bob_read_cost_metrics` tool + `bob-status` row) in the same PR. The substrate ships *because* its first consumer ships.
- **Auto-kill that inverts freeze-then-verify / no-premature-termination (the central doctrine risk):** carry X7's verified guard forward verbatim — cost telemetry is **DESCRIPTIVE ONLY, never an auto-kill gate** (nodes.json X7). A "$-budget reached → terminate session" gate would suppress findings before VERIFY/GRADE, inverting no-premature-termination exactly like the excluded `X-rej-1`/`X-rej-2` skips. Explicitly forbid any control-flow read of the cost record; it is a sink, not a brake.
- **Plane-O O-P1 (`--network none`) + hard-rule no-PII:** the in-session reconstruction is a **local-file parse only**; the ±5% reconciliation against the Anthropic billing surface is an **offline operator step**, never an egress from a hunt/OSS container (`repo-env.js` enforces `--network none`). Cost rows are integer/USD/model-ID aggregates run through `sensitive-material.js` redaction — never transcript bodies, tool arguments, or any operator identifier (the `vmihalis.tmd@gmail.com`-class data the project hard-rule bans), so no personal data is reused as tool input or written to a shared artifact.
- **Frontier-node safety/scope gate:** zero target interaction (observability, not an attack capability). The only scope surface is cross-session aggregation under `cross_session_read` — keep it metrics-only so one target's economics never carries another target's finding content into view.
- **Context-as-strategy (#2):** not applicable — solved by a deterministic index + bounded query, the doctrine-correct move, not by widening context.
