# C13 — Cross-target finding transfer
**Kind / Tier / Type:** C (Roof / capability) · Δ4 (frontier) · net-new   **Verdict from verification:** revise — anchor was incomplete and scope-risky. The dispatch primitive is not just `bob_start_wave` (which is single-domain, `mcp/lib/tools/start-wave.js:26`); transfer additionally needs `bob_init_session(targetB)` and a **hard scope-authorization gate enforced in the dispatch/authority layer, not the orchestrator prompt**. It rests on the claims subsystem (`claims.js` + `claim-clusters.js`), not the deleted `findings-index.js` (regression I6-was, commit `c02179c`).

## Spec (one-paragraph contract)
Given a *confirmed* finding-family on authorized target A (a `claims.js` claim whose status reached `verified` — `mcp/lib/claims.js:44`), C13 is the orchestrator routine that (1) queries the I13 cross-target transfer index for structurally-similar surfaces/symbols on **other targets**, (2) admits a target B only if B is on an **operator-owned authorized-targets allowlist** checked inside `authorizeBootstrap` (`mcp/lib/session-authority.js:926`), then (3) calls `bob_init_session(targetB)` and (4) `bob_start_wave(targetB, …)` with a *bounded* sibling-hypothesis brief slice. **Inputs:** a verified claim on A + I13 sibling hypotheses. **Outputs:** new scoped waves on authorized siblings that produce findings a triager recognizes as the same bug class on a different asset. **Predecessors:** I13 (Δ-new). **Enforced context budget:** sibling-hypothesis slice ≤5 default / 15 hard ceiling (the pattern the deleted `priors_slice` used, `docs/capability-hypergraph.md:570`), ≤500 tok/hypothesis; ≤3 auto-spawned siblings per confirmed finding (10 hard).

## What it does / holds
C13 holds no new persistent index of its own — it is the **consumer** that justifies I13. Its durable artifacts are the ordinary per-target session/claim/wave files that `bob_init_session` and `bob_start_wave` already write (`init-session.js:86`, `start-wave.js:36`). The one new piece of *state* it introduces is the **authorized-targets allowlist**: a project-level, operator-owned JSON document (modeled exactly on egress profiles at `.claude/bob/egress-profiles.json`, `mcp/lib/egress-profiles.js:19`) that the authority layer reads but no agent-callable MCP tool can write. The finding C13 produces that a triager recognizes: "the same vulnerability class confirmed on A reproduces on authorized asset B" — e.g. an IDOR pattern verified on target A's `/api/orders/{id}` re-confirmed against B's structurally-identical handler, each entering B's own freeze-then-verify pipeline as an independent claim.

## Failure mode it prevents
Two distinct ones. (1) **Missed scope-wide impact:** a verified bug on A that silently also affects B,C,D never gets transferred because nothing connects per-target claim stores — exactly the gap left when `findings-index.js`/`priors_slice` were deleted (`c02179c`); the cross-target prior now survives only as the orphaned `claim-clusters.js mergePriorClaimMatches` (`:115`, unconsumed). (2) **The dangerous failure C13 itself could cause — autonomous out-of-scope testing.** Today the *only* authorization gate is the human operator typing `/bob-evaluate <target>`; the bootstrap authority path self-authorizes **any** syntactically-valid public domain whose `target_url` is first-party (`session-authority.js:857-862` returns `allowedDecision` with no allowlist consulted). An auto-transfer routine that calls `bob_init_session(targetB)` would therefore bypass the human entirely and begin hitting an unauthorized third party — the machine-scale version of the kinky.nl account-creation incident. C13 exists to make that structurally impossible: transfer-originated bootstrap fails closed unless B is operator-pre-authorized.

## REVIEW (parishioner / real-target gate)
The honest closing gate is **engineering/fixture + a narrow real-target confirmation, long-horizon.** Engineering: on two targets the operator has *already* authorized and run independently, a verified claim on A drives C13 to spawn a scoped wave on B, and B's pipeline confirms the same bug class as its own claim — with the allowlist proven to block a third, non-authorized domain in the same run (this is the load-bearing safety assertion, not a finding count). The full "one confirmed finding transferred to a second authorized target produces a triager-accepted report" milestone is Δ4 and depends on having two structurally-similar authorized programs in flight at once — rare at operator volume, so treat it as long-horizon rather than a near-term blocker.

## Context budget
- Sibling-hypothesis brief slice: **≤5 default, 15 hard ceiling**; ≤500 tokens per compressed hypothesis tuple `(target_B, matched_surface_id, bug_class, similarity, source_claim_id)`.
- Auto-spawned siblings per confirmed finding: **≤3 default, 10 hard**.
- The transfer routine carries no raw artifacts from A — only claim ids + structural descriptors (dereference via I13), per context-as-strategy doctrine.

## Risks & doctrine compliance
- **Frontier safety/scope gate (primary).** This is the single highest-risk node in the graph: a capability that autonomously crosses target boundaries. The gate is mandatory, lives in the **authority layer** (`authorizeBootstrap`, on the dispatch path), is **fail-closed**, **operator-owned** (`.claude/bob/authorized-targets.json`, no MCP write tool), and **cannot be overridden by prompt text or the orchestrator**. Without it, C13 is the kinky.nl incident at machine scale.
- **Hard-rule no-PII.** Every spawned hypothesis brief restates the `/Users/memehalis/sec/CLAUDE.md` rule verbatim; tests on B use `bounty_temp_email` + synthetic identifiers; A's captured real data is never replayed against B.
- **auto-kill / freeze-then-verify.** No inversion: C13 only *spawns* hypotheses; each enters B's normal freeze-then-verify pipeline. The transfer routine must **not** auto-confirm or auto-suppress sibling hypotheses (mirrors the C15 no-auto-suppress invariant). The allowlist block is a *safety* gate, not a finding-suppression gate — doctrine-compliant.
- **index-without-consumer.** Avoided by construction: C13 *is* I13's first consumer; ship them together (`docs/capability-hypergraph.md:536`).
- **substrate-as-trajectory.** Avoided: C13 produces scope-wide findings, a real capability ship.
- **Plane-O O-P1.** This node's anchor is web-mode (`bob_init_session` refuses repos, `init-session.js:9-18`). Transfer across **repo/OSS** targets would route through `bob_init_repo_session` and inherit S15's `--network none` / init-never-clones invariants — out of this node's first slice; if added later the same allowlist gate must apply to the repo bootstrap class (`session-authority.js:91`).
