# S12 — Test-manifest + consumer-registration integrity
**Kind / Tier / Type:** S (slab-harden) / Δ1 / harden   **Verdict from verification:** revise + absorb (re-anchored; ~80% shipped in `test/mcp-test-discovery.test.js` + `scripts/check-stigmergy-coherence.js`; demoted X9 folded in; demoted from strict I9 blocker to same-PR fast-follow per WEAVE §5/§8).

## Spec (one-paragraph contract)
Close the **silent-drop class** at the CI layer: a guard test (or a brief/prompt-consumed field) can exist in the tree yet never execute / never be checked, so a producer regresses and CI stays green. **Inputs:** the test tree (`test/*.test.js`), `test/mcp-test-manifest.json`, the `package.json` test scripts, and the two stigmergy manifests (`mcp/lib/stigmergic-{producers,consumers}.js`). **Outputs:** (a) an extended discovery test that fails when any on-disk test file is run by *no* runner, and (b) a registered producer+consumer pair for the I9 reachability field-set so `check:stigmergy-coherence` fails if the brief renderer goes inert. **Predecessors:** S5 (SHIPPED — MCP tool registry + generators + manifest discipline). **Unlocks:** protects I9 and, by construction, every future guard test. **Context budget:** none added to any runtime brief — this is dev/CI-only infrastructure; the consumer manifest grows by exactly one frozen object (6→7 entries), the producer's `registered_consumers[]` by one id.

## What it does / holds
Two coupled CI invariants:

1. **Manifest-coverage invariant.** Today `test/mcp-test-discovery.test.js` only proves the *`mcp-`-prefixed* subset of the manifest is in sync — the discovery regex is `/^mcp-.*\.test\.js$/` (`mcp-test-discovery.test.js:17`) and the sync `deepEqual` runs over `manifest.filter(... .startsWith("mcp-"))` (`:24`, `:31`). The existence check (`:28`) and dup check (`:26`) cover all 104 manifest entries, but **nothing asserts that every `test/*.test.js` file on disk is actually executed by some runner.** S12 holds the stronger property: `on-disk test files == (manifest ∪ files named in any package.json test script)`.

2. **Consumer-registration invariant.** The reachability triage fields (`attack_vector`, `severity_ceiling`, `network_reachable`) are whitelisted into the evaluator brief at `assignment-brief.js:139-141` and named in the evaluator/orchestrator prompts (`:136-138`), but they are **not registered as a stigmergic consumer**. S12 registers the I9 producer (reachability stamping) and a consumer entry pointing at the brief whitelist, so `check-stigmergy-coherence.js` `checkAssertionA/B/C` (`:67/:92/:131`) treat a renderer that stops emitting them as a coherence violation.

## Failure mode it prevents
The exact regression WEAVE was written to explain, reproduced live this session: **a guard test exists but is wired to no runner.** `test/repo-target.test.js` holds the reachability guard tests at `:231-271` (`network_reachable`, `max_credible_severity_ceiling` assertions) — yet it is **not in `mcp-test-manifest.json`** (verified: `m.includes("test/repo-target.test.js") === false`) and appears in **no** `node --test` list in `package.json`. So `test:mcp` (`run-mcp-tests.js:30` spawns `node --test ...manifest`) never runs it, the I9 producer drop went undetected, and the file even rotted to a broken `require("../mcp/lib/hunter-completion.js")` (`repo-target.test.js:25`, the module was renamed to `agent-run-completion.js`) without any red CI. 37 `*.test.js` files are off-manifest; ~20 of them (incl. `dashboard`, `evaluator-spawn`, `graph-scheduler`, `lifecycle-advance`, `surface-index-authoritative`, `repo-target`) run **nowhere**. S12 turns each of those into a CI failure.

- **index-without-consumer:** S12 *is* the enforcement against this anti-pattern, so it must not become one. The new producer/consumer pair must point at **live** code — `checkAssertionB` (`:92-129`) refuses to register a consumer whose token doesn't resolve at the cited file. This is exactly why slice 5 is sequenced **after** I9 restores `mcp/lib/reachability.js`/`buildRepoInventory`; registering before the producer exists would (correctly) red CI. *(assumed:* I9 lands `mcp/lib/reachability.js`; if I9 keeps the producer inline in `repo-target.js`, point `trace_shape_ref`/`token_or_regex` there instead.)*
- **auto-kill that inverts freeze-then-verify / no-premature-termination:** S12 adds **CI gates, never a finding-killer** — it cannot suppress or downgrade any candidate claim, so freeze-then-verify is untouched. The one auto-kill risk is a *flaky false-positive* against legitimately dedicated-runner tests (`cli`, `package`, `install-smoke`, `prompt-contracts`, etc.); slice 1's "manifest ∪ scripts" union is designed to allowlist those by construction rather than denylisting them by hand.
- **substrate-as-trajectory / context-as-strategy:** N/A — no new brief content, no scheduling, no trajectory state.
- **Policy:** no target interaction, no network, no PII — Plane-O O-P1 (`--network none`) and the hard no-PII rule are not engaged. No frontier/safety gate applies (pure dev-tooling, no agent spawn).
