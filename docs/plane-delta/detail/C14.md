# C14 — Proof-carrying disclosure (bundle)
**Kind / Tier / Type:** C / Δ2 / net-new   **Verdict from verification:** SPLIT — the full machine-checkable bundle is the net-new Δ2 node (sole hard predecessor C10); the CWE/CVSS/refs metadata splits off as `C14-lite` (adopt, Δ1, no predecessor). The Δ3 `I12` predecessor is DROPPED (tier inversion): CVSS/CWE assemble from final-verifier `severity` + `attack_vector`, not root-cause families.

## Spec (one-paragraph contract)
For each **final-reportable** finding (`reportable === true` in the final verification round, `verification-round-store.js:121-122`), C14 produces a content-addressed `ProofBundle` — an MCP-owned, hash-bound artifact carrying one or more of {replay script runnable via `bob_repo_docker_run`, SC invariant via `runInvariantForFinding`, differential control row from C10}. **Inputs:** `target_domain`, `finding_id`, the C10 differential evidence row `{vuln_run_id, control_run_id, control_kind, verdict}`, and (for OSS) the `/work` replay run id from `bob_repo_docker_run`. **Outputs:** `proof-bundles.json` (+ `.md` mirror), each pack keyed by `finding_id` with a `bundle_hash` and a final-verification binding triple; a new `proof_bundle` report section (compose-report `SECTION_KINDS`) and a `proof_bundle:` evidence-ref prefix. **Predecessors:** C10 (Δ-new, blocking). **Unlocks:** file-ready disclosures, C17. **Context budget:** bundle text fields inherit evidence.js caps (`replay_summary` ≤ 2000, snippet ≤ 4000, any JSON value ≤ 8000 chars); no raw body ≥ `LARGE_BODY_THRESHOLD_BYTES` (262144, `paths.js:433`) — reference an import handle instead.

## What it does / holds
A `ProofBundle` is the *re-executable* counterpart to the existing **descriptive** evidence pack (`evidence.js:199-247`: `replay_summary`, `representative_samples`, `report_snippet`). Where the evidence pack says "this is what happened," the bundle says "run this and it happens again." It stores, per `finding_id`:
- `bundle_kind ∈ {replay_script, invariant, differential}` and an `artifacts[]` array. For OSS: a bounded replay command + the `/work` `run_id`/`run_hash` returned by `bob_repo_docker_run` (mounts `/src` read-only, `/work` read-write, `--network none` by default — `repo-docker-run.js:30-34`). For SC: the `invariant-runs.jsonl` `run_hash` from `runInvariantForFinding` (shipped, `capability-hypergraph.md:345`). For the differential proof: the C10 row `{vuln_run_id, control_run_id, control_kind, verdict}` proving residual / self_patch / pre_introduction (C10 anchor: `write-evidence-packs.js` + `evidence.js`).
- A `bundle_hash` (content-addressed, timestamps zeroed) and the final-verification binding triple `{verification_attempt_id, verification_snapshot_hash, final_verification_hash}` (reuse `evidenceBindingForFinal` / `assertEvidenceMatchesFinal`, `evidence.js:356-388`).

The finding a triager recognizes: a disclosure where the maintainer runs one command (`--network none`, sandboxed) and independently observes the bug — and, via the C10 control, sees the same command go quiet against the patched tree. The bounded run id surfaces in `report.md` through the new `proof_bundle` section and a `proof_bundle:<finding_id>` evidence-ref that the compose-report provenance gate (`compose-report.js:191-253`) validates exists.

## Failure mode it prevents
Unreproducible disclosures. Today the report carries prose + a `replay_summary` string (`reporter.md:30` already gestures at "the bounded command/status/run ID from the evidence pack") but nothing the recipient can mechanically re-run, and nothing proves the *control* (that the bug is absent on the fix or pre-introduction commit). That gap is exactly what gets a real finding triaged as "informational / can't reproduce" — the live OSS campaign's most expensive failure mode. C14 makes "machine-checkable" a stored, hash-bound property rather than a claim in prose.

## REVIEW (parishioner / real-target gate)
A maintainer/triager, given only the bundle, runs one sandboxed command (`bob_repo_docker_run`-shaped, `--network none`) and independently reproduces the finding, then runs the C10 control and observes it go quiet on the patched/pre-introduction tree. The honest closing outcome: **a live-campaign disclosure (e.g. netatalk / openexr / libcupsfilters) accepted with its runnable PoC bundle attached** — i.e. the recipient confirms reproduction from the bundle rather than re-deriving from prose. This is reachable now on the OSS disclosure campaign; it is not engineering/fixture-only.

## Context budget
No new long-lived subagent. If a bundle-assembly step is delegated (e.g. evidence-agent composing the replay artifact), its brief slice is capped at the evidence caps above (≤ 2000 / 4000 / 8000 char fields) and must reference the `/work` run id, not inline logs. The `proof-bundles.json` slice surfaced into any brief follows the existing reachability-field budget pattern (`attack_vector: 40`, `wave-brief-derivation.js:132`) — bounded, hash-keyed, dereferenced on demand, never inlined whole.

## Risks & doctrine compliance
- **Substrate-as-trajectory:** avoided — C14 ships with two live consumers (the `report.md` `proof_bundle` section and C17). Do not merge the writer without at least the report-wiring slice (#5).
- **Index-without-consumer / context-as-strategy:** N/A as an index, but enforced in spirit — the bundle is referenced by `bundle_hash` and dereferenced via tool, never widened into context.
- **Auto-kill / freeze-then-verify / no-premature-termination:** HARD invariant — a bundle that *fails to build or fails to replay* must NEVER suppress or drop the underlying finding from the reportable set (mirror C15's no-auto-suppress rule and C9's cap-not-kill). C14 attaches to findings *after* final verification (`reportable === true`); it is downstream of freeze-then-verify, never a gate on it.
- **Plane-O O-P1:** replay execution is `--network none`, `/src` read-only, `/work` read-write, init never clones (`repo-docker-run.js:30-34`); the bundle stores the O-D6-matched `image_tag` so re-execution (C17) is reproducible without re-fetch.
- **No-PII hard rule:** every bundle text field passes `validateNoSensitiveMaterial`; replay scripts must use synthetic identifiers / `bob_temp_email`, never operator data — the script is shipped to a maintainer, so this is load-bearing.
- **(assumed)** C10's differential row `{vuln_run_id, control_run_id, control_kind, verdict}` is not yet in code (C10 is the net-new predecessor); the bundle's `differential` kind is specified against C10's declared anchor shape, not a verified struct. **(assumed)** `attack_vector` is the I9 reachability field, currently inert (producer dropped per `regressions[].I9-was`); until I9 is restored, C14-lite's CVSS must degrade to reporter-supplied / `AV:N` rather than block — this keeps the Δ1 ride-along honestly Δ1 (no hidden I9 dependency).
