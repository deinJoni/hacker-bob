## Spec (one-paragraph contract)
**Inputs:** a detected `oss_native_code` C/C++ surface (`capability-packs.js:97`) with `repo-env.json` produced by `bob_repo_prepare_env`, an IP8 per-target seed store, and the operator's build decision. **Outputs:** (a) a `role: "fuzz"` entry actually emitted by `recommendedCommandsFor` (`repo-env.js:207`) carrying a sanitizer-matrixed `clang -fsanitize=…,fuzzer` recipe; (b) a session-derived Docker image that links aarch64 compiler-rt so the recipe builds on Apple Silicon; (c) a bounded `fuzz_stats` scalar block on each `repo-command-runs.jsonl` row enabling plateau detection without reading guarded stdout. **Predecessors:** IP8 (Δ-new, emits the fuzz role + seed store) → which itself rests on S15 (Δ-new OSS-container substrate). **Unlocks:** native-code findings, and crucially **unblocks C9's harness dependency** — when the harness can link and run, the record-time proof gate in `evaluator.md:64` stops downgrading HIGH native findings to `blocked_harness_runs[]`. **Enforced context budget:** all new brief surface lands inside the existing `repo_workflow` slice cap of **2048 tokens** (`assignment-brief.js:397`); the per-run plateau signal is a structural scalar block bounded by construction like the X-P9 `stdout_first_line` retrofit (`repo-env.js:1326-1335`).

## What it does / holds
Concretely, C12 turns Bob's *declared-but-dead* `fuzz` role into a working coverage-guided pass on the one host the operator actually runs (aarch64 macOS), and makes the run's progress legible to the orchestrator as a bounded query rather than a context dump.

- **Emit the fuzz role.** Today `recommendedCommandsFor` (`repo-env.js:207`) emits `build`/`test`/`compose` only; the C/C++ branch (`:314-329`) emits a single `compose`-role staging recipe and **no** `fuzz` entry, even though `"fuzz"` is a valid role (`RECOMMENDED_COMMAND_ROLES`, `repo-env.js:137-143`, slot `:141`) and the validation loop already accepts it (`repo-env.js:569-572`). C12 adds the fuzz-role command(s).
- **Sanitizer matrix.** The emitted fuzz recipe carries a small deterministic matrix — ASAN+UBSAN+libFuzzer as the default pair (`clang -fsanitize=address,undefined,fuzzer -g -O1`), MSAN gated behind an instrumented-libs precondition *(assumed: MSAN needs instrumented deps and is usually impractical — emit it only as an optional second variant, never as the default)*.
- **aarch64 compiler-rt fix.** The generated Dockerfile (`buildDockerfileBob`, `repo-env.js:351`) installs `C_DEFAULT_APT_PACKAGES` (`:120-127`) which ships `clang` but **not** `libclang-rt-18-dev`; on aarch64 the link fails because `libclang_rt.{fuzzer,asan}-aarch64.a` are absent (the exact blocker documented in `reference_bob_oss_libcupsfilters_fuzz_image` and `feedback_bob_oss_aarch64_libfuzzer_harness`). C12 adds compiler-rt to the apt set when a fuzz harness is detected, baking it into the **session-derived** image.
- **Structure-aware harness hints.** A bounded knowledge slice in the fuzz lens telling the evaluator how to shape seeds for length-prefixed/framed formats (e.g. "nal_fuzzer wants 4-byte big-endian length-prefixed NAL units"; seed from `/src/testdata/*`). This is hint scaffolding for LLM-authored harnesses, not an automated grammar fuzzer (honest scope).
- **Plateau detection (index it holds).** A `fuzz_stats` block `{cov, ft, exec_per_s, corpus_size, crashes}` parsed from libFuzzer's progress lines at run completion and stored on the JSONL row, mirroring the X-P9 distilled-summary pattern (`repo-env.js:1300-1335`). The bounded query it enables: "did coverage flatline across the last N runs?" → signal to switch to structure-aware seeds, **without** reading the read-guarded `repo-runs/<run_id>.stdout`.

The finding a triager recognizes: an ASAN/UBSAN crash with a minimal reproducer and a real non-dry-run `bob_repo_docker_run` replay matching `repro_command` (the exact bar `evaluator.md:64` sets for HIGH native findings).

## Failure mode it prevents
The specific bad outcome: a real coverage-guided native bug is **never found, or found but downgraded**, because the harness won't link on the operator's host. Without C12 a native C/C++ surface either (a) falls back to a "dumb gcc+ASan mutation loop" the memory explicitly warns against, or (b) records `blocked_harness_runs[{kind:"sanitizer_unavailable"}]` and stays `surface_status: partial` (`evaluator.md:59,81`), which then forces the record-time proof gate (`evaluator.md:64`) to refuse the HIGH and leave a static-only claim. That refusal silently overrides C9's grade-time severity lift (C9 node `note`). C12 removes the linker block at its root so the ladder doesn't stall at Δ1 on a toolchain gap (the exact "gate-resourcing risk" called out in `capability-hypergraph-delta.md:112,119`).

A second, subtler failure it prevents: **plateau-as-auto-kill**. Naively, "coverage stopped climbing" invites auto-terminating the surface. C12's plateau signal is *descriptive only* and must never mark a surface complete — preserving freeze-then-verify and the `hunting.md` rule that "existing test passes / plateau reached" is a starting point, not a termination condition.

## Risks & doctrine compliance
- **Index-without-consumer:** clean. The `fuzz_stats` index has an immediate consumer (plateau-read in the same fuzz lens); IP8's seed store is consumed by this node's recipe. Neither ships speculatively (anti-pattern #4, capability-hypergraph.md:536).
- **Substrate-as-trajectory:** clean. C12 is a Roof capability that produces findings and pulls IP8/S15; it is not substrate shipped for its own sake (anti-pattern #1).
- **Context-as-strategy:** clean. Plateau is solved with a 5-scalar index, *not* by widening context to read guarded stdout (anti-pattern #2, capability-hypergraph.md:534).
- **Auto-kill inverting freeze-then-verify / no-premature-termination:** the headline risk. Plateau detection must be wired as a *descriptive switch-strategy signal only* — it must never set `surface_status: complete`, never auto-write `blocked_harness_runs`, and never short-circuit the verification spine. The MCP `complete`-gate on native surfaces still requires a recorded finding or a cited bypass attempt, and rejects `complete` when `blocked_harness_runs` is non-empty (`evaluator.md:81`); C12 does not touch that gate. This directly honors `evaluating.md`/`hunting.md`: "existing test passes / plateau reached" is a starting point, not a termination condition.
- **Plane-O policy:** **O-P1** (`--network none` at fuzz run time, `repo-env.js:407`) and **O-D6** (no off-session-tag image, `repo-env.js:1124-1135`) are the two constraints that *force* the design — compiler-rt must bake into the session-derived image at build time, not a side `:fuzz` tag, otherwise `repoDockerRun` throws `image_tag_mismatch`. **O-P3** cap-drop/no-ENV-secrets is preserved. **Hard-rule no-PII:** not applicable to OSS container runs (no target HTTP signup surface), but the `validateNoSensitiveMaterial` guard on the new JSONL field keeps the rule's spirit.
- **Honest-severity (per `feedback_oss_severity_honest_gating`):** when a stubbed `config.h` is used to make a harness compile, the finding must not overclaim E2E reachability; C12's recipe defaults to the real `./configure` path and only falls back to a stub with an explicit reachability caveat.
