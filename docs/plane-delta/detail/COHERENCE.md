## Check 2 — Conflicting file/function contracts

- **S13 ↔ X6 (`wrapUntrusted` ABI — 5 mismatches, strongest finding):**
  - return field: S13 returns `{ text, … }`; X6 calls `wrapUntrusted(value).fenced` → **undefined**.
  - constant name: S13 `FENCE_OVERHEAD_CAP`; X6 `UNTRUSTED_FENCE_OVERHEAD_CHARS`.
  - constant value: S13 cap **256**; X6 budget **160**.
  - standing note: S13 exports `UNTRUSTED_DATA_SYSTEM_NOTE` (≤512); X6 defines its own `untrusted_content_policy` (≤256) and never consumes S13's export.
  - signature: S13 requires `wrapUntrusted(content, { label })`; X6 calls `wrapUntrusted(content)` with no label.
  **Fix:** pin one contract in S13 and make X6 import it verbatim (recommend S13 as source of truth: `{ text }`→rename to `fenced` or update X6; single overhead constant + value; X6 uses `UNTRUSTED_DATA_SYSTEM_NOTE`; X6 supplies the slice label).

Field names conflict: IP7 `{tool_name, severity_signal, artifact_uri, start_line, result_sha256}` vs I10 `{tool, severity, location(path:line:col), finding_hash}`. C11 expects I10's shape but as `{rule_id, file, start_line, message, cwe}` (yet another variant). Topology says "I10 **absorbs** IP7." **Fix:** merge IP7 into I10 (one `parseSarif`, one canonical JSONL path, one record shape); align C11's expected `file`/`start_line` to the chosen shape (or add a `file`/`start_line` accessor).
- **I12-plumbing ↔ I12 (`context.ossTechniquePacks` double-producer, conflicting content):** Both claim to be the producer that "**finally** gives the orphaned `:403` consumer a producer." I12-plumbing fills it with the **7 OSS technique packs** (`OSS_TECHNIQUE_PACKS`, partitioned by lens); I12 fills it via `suggestFamiliesForSurface(...)` with **root-cause families**.
