**Kind / Tier / Type:** IP · Δ3 · net-new   **Verdict from verification:** ✏️ revise — narrowed to **ingest-only** (the runner half already ships), predecessor corrected S6→**S15**, and explicitly **may merge into I10** (WEAVE.md:16; nodes.json id `IP7`). ## Spec (one-paragraph contract)
**Inputs:** the captured output of an already-completed `bob_repo_docker_run` semgrep/trivy invocation — a `run_id` resolving to `repoRunsDir(domain)/<run_id>.stdout` (when SARIF is emitted to stdout) or a path under `repoWorkDir(domain)` (when emitted to `/work/<tool>.sarif`). **Output:** a deterministic, redacted, content-addressed JSONL of normalized static-analysis results (`static-analysis-results.jsonl`), one record per SARIF `results[]` entry, handed to **I10** for dedup + lead emission. **Predecessors:** S15 (Δ-new register of the Plane-O container substrate). **Unlocks:** I10. **Enforced context budget:** the ingest tool never streams raw SARIF into LLM context — it reads the captured host-side artifact, normalizes on the host (MCP-owned), and returns only a bounded summary (≤2k tokens); per normalized record ≤300 tokens; parse capped at `MAX_RESULTS` per run. IP7 introduces **no evaluator brief slice and no subagent** — I10 owns the brief-facing slice that feeds C11.

## What it does / holds
For each SARIF 2.1.0 result it stores a normalized tuple mirroring the S6 record shape (`static-artifacts.js:154` `normalizeStaticArtifactRecord`):
`{ tool_name, tool_version, rule_id, level→severity_signal, message (redacted), artifact_uri (repo-relative file), start_line, end_line, fingerprint, result_sha256, source_run_id }`.
SARIF→tuple field map: `runs[].tool.driver.{name,version|semanticVersion}` → `tool_name/tool_version`; `runs[].results[].ruleId` → `rule_id`; `results[].level` (`error|warning|note`) → `severity_signal`; `results[].message.text` → `message`; `results[].locations[0].physicalLocation.artifactLocation.uri` + `region.startLine/endLine` → `artifact_uri/start_line/end_line`; `results[].partialFingerprints` (or a computed hash when absent) → `fingerprint`. The **bounded query it enables** (consumed by I10): "give me the deduped static results for file X / rule R" — turning a multi-MB scanner dump into a `(file:line → rule → signal)` lookup. A result is a **lead seed**, never a finding: `severity_signal` is descriptive only.

## Failure mode it prevents
Two concrete bad outcomes. (1) **Context blow-out / hallucinated correlation** — without normalization the only way to use scanner output is to paste raw semgrep/trivy text (or a multi-MB SARIF blob) into an evaluator, diluting attention and inviting fabricated cross-file links (the same failure I1 was built to stop, `capability-hypergraph.md:69`). (2) **Scanner output masquerading as findings** — a SARIF `error` is a hint, not a confirmed bug; IP7 forces every result into a deduped *lead* tuple so it can only ever enter the pipeline through I10→C11→`lead-promotion.js`, never as an auto-finding and never as an auto-skip.

## REVIEW (engineering)
- **Tests** — new `test/sarif-ingest.test.js`: (a) parse a real SARIF 2.1.0 `semgrep.sarif` + `trivy.sarif` fixture → expected normalized tuples; (b) **determinism** — same SARIF bytes → identical records → identical `result_sha256` across runs; (c) **malformed** — truncated/garbage JSON and a missing `locations[]` row → warn-and-skip, no throw, valid rows still ingested; (d) **empty** — `runs:[]`/`results:[]` → `[]`, zero leads (no orphan/false index entry); (e) **cap** — >`MAX_RESULTS` results truncate deterministically; (f) **redaction** — a `region.snippet`/`message` carrying a fake `AKIA…`/URL credential is redacted before persistence.

## Risks & doctrine compliance
- **index-without-consumer (real risk).** IP7 is inert until I10 consumes it; ship them together or merge `sarif-ingest.js` into `static-analysis-index.js` (the explicit "may merge into I10" verdict). Do not land IP7 alone.
- **substrate-as-trajectory.** SARIF results are substrate, not trajectory — they become leads via I10, drive evaluators via C11, and only become findings through the normal verify path. Severity stays a `severity_signal`, never a verdict.
- **auto-kill inversion (freeze-then-verify / no-premature-termination).** A clean/empty scan must **never** suppress hunting and a SARIF `error` must never short-circuit verification — this is the X-rej-1 line ("adopt the SIGNAL never the kill", `nodes.json` excluded_by_doctrine). IP7 only *adds* leads; it has no skip/prune authority.
- **Plane-O O-P1 tension (flag, don't fix here).** `semgrep --config auto` fetches its rulepack over the network and `trivy fs` wants to pull its vuln DB — both conflict with the `--network none` default (`repo-env.js:838`). IP7 ingests whatever SARIF an **offline-or-egress-gated** run produced; making the runner work offline (baked rules/DB at `prepare_env`, or egress-profile `allow_network`) is the runner/C12-image concern, not IP7's.
- **CodeQL gap (honest scope).** The node title says "semgrep/trivy/**CodeQL** runner," but CodeQL is explicitly *deferred* — not a seed pack (`cli-tool-packs.js:213`). IP7's parser is SARIF-generic (CodeQL emits SARIF natively), but no CodeQL pack ships today; treat CodeQL as future once a pack lands.
- **no-PII hard rule.** Mandatory redaction of `message`/snippet bytes before persistence (reuses `redactStaticArtifactContent`); no operator identifiers are ever inputs here (the tool parses scanner output, not user data).
