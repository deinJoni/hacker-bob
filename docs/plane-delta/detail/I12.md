**Kind / Tier / Type:** I (index) / Δ3 / adopt (extend SHIPPED I5 corpus shape)   **Verdict from verification:** SPLIT — `WEAVE.md:14` carves off **I12-plumbing** (Δ1, `harden`) as the cheap ride-along that wires the orphaned `context.ossTechniquePacks` consumer (`assignment-brief.js:403`); only the named families + worked witnesses remain real Δ3. ## Spec (one-paragraph contract)
**Inputs:** an OSS surface object (task lens, language/stack, source/sink hints from I10 SARIF + I9 reachability) and, at authoring time, public CVE/commit witnesses. **Outputs:** a frozen, content-addressable corpus of *named root-cause families* — each `{ id, family, name, description, lens_affinity[], source_sink_signature[], witness{} }` — plus a bounded query `suggestFamiliesForSurface(surface, { lens, limit })` returning the families whose `lens_affinity`/`source_sink_signature` match the active lens, with a one-line witness per family. **Predecessors:** I5 (SHIPPED, shape donor), I12-plumbing (Δ-new, makes content reachable). **Enforced context budget:** the rendered family slice rides the existing `technique_packs` brief slice capped at **8192** chars/tokens (`assignment-brief.js:403`, `:261`); per-family brief form ≤ **240 chars** (the existing `TECHNIQUE_SUMMARY_ITEM_MAX_CHARS`, `technique-packs.js:385`); full witness detail is an on-demand read, never auto-inlined — exactly as I5 keeps `foundry_test_template` out of the brief.

## What it does / holds
**Stores.** A new corpus module modeled byte-for-shape on `mcp/lib/invariant-template-corpus.js` (`TEMPLATES` frozen array → `TEMPLATES_BY_CLASS` Map → `SUPPORTED_CLASSES`, `invariant-template-corpus.js:3-118`). Each record names a *root-cause family* rather than a Foundry template: a recognizable source→sink shape, its `lens_affinity` (reusing the OSS pack vocabulary `taint_trace`/`fuzz_run`/`code_surface_scout`, `technique-packs.js:89-92`), a `source_sink_signature[]` (the I5 `parameter_slots[]` analogue — the controlling fields a hunter must locate), and a **worked witness** `{ project, cve_or_commit, file_symbol, controlling_fields, impact }`.

This refactors the single prose blob in the `oss_native_code` pack summary (`technique-packs.js:89-99` — "bounds checks, integer truncation… double-free/use-after-free… map data path from packet/file/API input to exact parser") into *discrete, queryable families*. It adds the **2 families absent** from that blob:
- **`crypto_ordering` (MAC-then-decrypt / decrypt-before-authenticate):** ciphertext or attacker-controlled bytes are decrypted/parsed before the MAC/signature is verified → padding-oracle / plaintext-recovery / forged-data processing. Signature slots: `decrypt_call`, `mac_verify_call`, `ordering_between_them`.
- **`validate_vs_consume` (parse-then-validate / consume-before-bound):** a length/size/index field from untrusted input is *used* (alloc, `memcpy`, loop bound) before the bounds check runs. Directly generalizes the disclosure-campaign witnesses already in operator memory: rpcbind `rpcbaddrlist()` unbounded XDR string → `buf[128]` stack overflow; netatalk afpd Spotlight RPC length consumed before validation (heap OOB); libtirpc `__xdrrec_getrec` heap OOB write; LibreDWG `read_literal_length` 0xFFFF loop. Signature slots: `length_field`, `consuming_op`, `bound_check_site`, `entry_to_consumer_path`.

**Bounded query it enables.** Given a routed OSS surface + lens, return ≤`limit` families with one witness line each — so a hunter (and C11) pattern-matches a concrete source→sink shape instead of re-deriving the bug-class taxonomy per target. The finding a triager recognizes (emitted downstream via C11): *"attacker-controlled length `<field>` at `<file:symbol>` is consumed by `<memcpy/alloc>` before the bounds check (validate_vs_consume), reachable from `<network/file entry>` (I9); witnessed by `<CVE>`."*

## Failure mode it prevents
Two. (1) **Re-deriving the bug-class taxonomy per OSS target** — the I5 compounding rationale (`capability-hypergraph.md:174-175`, "Templates compound: corpus grows, per-audit subagent gets cheaper") applied to native-code/crypto families. (2) **Blind spots in the shipped `oss_native_code` blob:** that summary enumerates memory-safety classes but names *neither* crypto-ordering *nor* the parse-then-validate ordering bug as first-class shapes, so hunters miss MAC-then-decrypt entirely and treat each length-before-bound bug as a one-off. Naming the families with witnesses turns "go look for memory bugs" into "find this exact source→sink ordering, here is the CVE that proves it."

## Risks & doctrine compliance
- **index-without-consumer (the live risk here):** I12's content is exactly the `context.ossTechniquePacks` orphan (`assignment-brief.js:403`). Mitigation is structural — I12-plumbing is a hard predecessor and slice 4 registers the producer/consumer in the stigmergic manifests, so CI (`check-stigmergy-coherence.js`, the folded X9 / S12 gate) fails if the consumer goes orphaned again. **Do not ship I12 content without I12-plumbing.**
- **substrate-as-trajectory:** avoided — the corpus is a passive queryable index, not a sequencer; it does not drive control flow, only foregrounds families for a lens.
- **context-as-strategy:** avoided — families compress (replace a prose blob with discrete, witness-backed shapes) rather than widen context; the 240-char/8192-char caps enforce this.
- **auto-kill / freeze-then-verify inversion:** N/A — an index never terminates a hunt or suppresses a finding; it only suggests. No premature-termination surface introduced.
- **Tier discipline:** C11→I12 stays `augment` (non-blocking); the C14→I12 edge was dropped to avoid Δ2-blocks-Δ3 inversion (`WEAVE.md:39`).
- **Policy:** witnesses are public CVEs/commits and the operator's own (public) disclosure findings — **no PII, no operator personal identifiers** in any record (hard-rule). OSS scope only; no Plane-O `--network` interaction (the corpus is static authoring data, not a container action).
