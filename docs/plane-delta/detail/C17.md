# C17 — Proof-carrying regression sentinel
**Kind / Tier / Type:** C / Δ4 / net-new   **Verdict from verification:** ➕ new (verification-surfaced, CRITIC §2 — "cheap composition nobody captured"). Kept; no anchor revision needed, but tier honesty added: cheap in *new* code, expensive in *prerequisites* (see Predecessors).

## Spec (one-paragraph contract)
**Inputs:** a confirmed-AND-disclosed-AND-fixed finding that already carries a C14 proof-carrying bundle (replay script / invariant / differential control), the recorded post-fix baseline (the bundle's content-addressed replay artifacts + the "bug no longer fires" verdict), and a stream of new commit/CVE events from IP9. **Outputs:** an append-only, MCP-owned `regression-alerts.jsonl` row per re-fire, plus a bounded re-fire summary the orchestrator surfaces to the operator/disclosure campaign — a *signal*, never an auto-finding. **Predecessors:** C14 (bundle, Δ2-new), IP9 (commit/CVE feed, Δ4-policy-new), S1 (content-addressed evidence, SHIPPED). **Enforced context budget:** re-fire frontier in orchestrator ≤2k tokens; per-sentinel record ≤500 tokens; per-re-fire triage subagent ≤8k; replay stdout/stderr never enters context raw — only sha256 + bounded stderr tail + differential verdict (mirrors `repo-env.js` capture-file-with-hash discipline).

## What it does / holds
**Stores** (`regression-sentinels.jsonl`, one row per disclosed-and-fixed finding): `{ finding_id, c14_bundle_ref, fix_commit, fixed_version, replay_command_hash, replay_argv_hash, postfix_stdout_hash, postfix_stderr_hash, postfix_verdict:"does_not_fire", control_kind, applies_to[paths|symbols] }`. The replay keys are the **deterministic** `commandHash`/`argvHash` from `repo-env.js:1178-1179` — NOT `run_id`, which is timestamp+random (`generateRunId`, `repo-env.js:775`) and therefore unusable as a re-fire key.

**The bounded query it enables:** given a new HEAD commit from IP9, "which sentinels' `applies_to[]` overlap the changed files, and of those, which re-fired?" — answered by replaying each applicable C14 bundle once via `bob_repo_docker_run` (tool `mcp/lib/tools/repo-docker-run.js`, `dry_run` default true; `--network none` by default per `repo-env.js:1188`) against the new commit, then comparing the run's captured `stdout_hash`/`stderr_hash` + differential verdict against the recorded post-fix baseline.

**The finding a triager recognizes:** "CVE-XXXX's fix (commit A) regressed at commit B — here is the same machine-checkable bundle from the original disclosure, content-hash-matched, firing again." A regression of an accepted CVE is itself a file-ready advisory; this serves the operator's live campaign directly ("is the netatalk/openexr/libnfs fix still holding?").

## Failure mode it prevents
A vulnerability that Bob already found, disclosed, and got fixed silently **regresses** in a later upstream commit and nobody notices until it is re-exploited in the wild — the disclosure campaign goes stale because tracking "is each filed fix still holding?" is manual and unbounded. Secondarily, it prevents burning a full re-hunt (tokens + wall-clock) to rediscover a regression that a cheap, deterministic bundle replay catches for free.

## REVIEW (engineering)
- **Tests** (`test/regression-sentinel.test.js`): (a) re-fire keys on `command_hash`/`argv_hash`, NOT `run_id` (assert two runs of the same bundle with different `run_id`s but identical command produce identical re-fire keys); (b) a seeded revert-commit fixture → exactly one `re_fired` alert; (c) a non-touching commit → zero alerts; (d) malformed/missing C14 bundle → reject at register, no partial row; (e) "fix_holds" path persists nothing to the alerts ledger and **suppresses nothing** elsewhere.
- **Context-budget compliance:** re-fire frontier ≤2k; per-sentinel row ≤500 tokens; per-re-fire triage subagent ≤8k; raw stdout/stderr excluded from context (sha256 + bounded tail only).
- **Determinism / content-hashing:** sentinel rows and re-fire verdicts canonical-hashable (`sha256Hex`); same (bundle, commit) → same verdict → same hash. Reuse `normalizeArtifactHashes` (`verification-round-store.js:81`) value/key regexes for any artifact-hash map persisted.
- **Failure-mode coverage:** orphan (sentinel whose C14 bundle was deleted → register refuses, run skips with `indeterminate`); partial (alert append crashes mid-write → `withSessionLock` + atomic JSONL append, no half-row, S2); malformed (replay returns non-zero/timeout → `indeterminate`, never silently `fix_holds`).
- **Slab invariants preserved** (`capability-hypergraph.md:57`): **S1** — sentinel rows + replay captures content-addressed; **S2** — register/append via session-locked MCP writes, no state advance without manifest discipline; **S3** — a bundle containing a stateful/destructive step must respect its replay-lease typing on re-execution (no double-fire across commits); **S7** — no credentials in any sentinel row or alert summary. Plus Plane-O **O-P1** (`--network none`, init-never-clones; `repo-env.js:1188`) and **O-P3** flag set on every replay.

## REVIEW (parishioner / real-target gate)
**Honest gate is long-horizon and engineering/fixture-first**, because (a) real regressions of accepted fixes are rare and (b) C17 is blocked on the full C14 spine + policy-gated IP9. Near-term closing gate: against the operator's own campaign (e.g. a netatalk/openexr/libnfs fix that was accepted), register one sentinel, feed a sequence of real later upstream commits, and confirm the sentinel deterministically reports **fix_holds with zero false re-fires** across all of them; then feed a *synthetic revert* of the fix commit and confirm **exactly one** `re_fired` alert that maps cleanly to a file-ready regression advisory. The real-world milestone (a genuine upstream regression caught in the wild) is explicitly deferred behind IP9 shipping.

## Risks & doctrine compliance
- **Auto-kill / freeze-then-verify inversion (primary risk).** A `fix_holds` result must **never** auto-close, auto-suppress, or silently drop a tracked disclosure — and a `re_fired` result is a **signal only**: it flows back through the normal freeze→verify→grade pipeline before it can ever become a reportable finding. This is the inverse of C15's invariant ("a fixed-PoC patch must never auto-suppress the pre-patch finding"); encode it as a hard invariant in `run-regression-sentinel.js`. Never auto-submit a regression advisory — IP9's policy gate + two-turn human gate govern dispatch.
- **Index-without-consumer.** The alerts ledger must reach a human surface (operator note / dashboard) and have its consumer field-pair registered in `stigmergic-consumers.js` (S12). The I9 regression — wired-but-inert consumers — is exactly the failure mode to avoid here.
- **Substrate-as-trajectory.** A re-fire is evidence to verify, not a trajectory to act on autonomously.
- **Context-as-strategy.** No raw replay output is loaded; correctness rests on content hashes + verdicts, not on stuffing logs into context.
- **Policy / hard-rule.** Replay runs `--network none` in the Plane-O sandbox (no live-target interaction, no PII egress); the only network touch is IP9's feed pull, which is external + policy-gated. `validateNoSensitiveMaterial` scrubs every persisted row; alert prose must not embed operator identifiers (hard-rule no-PII).
- **Frontier safety/scope gate.** As a Δ4 frontier node, C17 acts only on **already-authorized, already-disclosed** findings within their original program scope; it introduces no new target surface (that is C13's concern, explicitly out of scope here).
- **(assumed)** that C14's bundle format exposes a stable, re-executable replay handle and a differential `control_kind` verdict suitable for hash-comparison — C14 is Δ2-new and unbuilt, so this contract must be fixed when C14 is detailed; C17's comparator depends on it.
