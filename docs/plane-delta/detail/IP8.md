## Spec (one-paragraph contract)
**Inputs:** the bound OSS repo at `/src` (read-only) already walked once by `buildRepoInventory` (`repo-target.js:752`, capped at 50k files by `walkRepo:526`), plus the detected language profile from `recommendedCommandsFor` (`repo-env.js:207`). **Outputs:** (a) a bounded, content-addressed **seed-corpus section** folded into `repo-inventory.json` ‚Äî discovered seed directories (`testdata/`, OSS-Fuzz `*_seed_corpus[/.zip]`, `corpus/`, `seeds/`) with per-dir file counts, total bytes, and a `seed_corpus_hash`; (b) the previously-dead **`fuzz`-role command** emitted into `recommended_commands[]` of `repo-inventory.json`'s sibling `repo-env.json`, pointing a libFuzzer/ASAN run at the staged seed dirs. **Predecessors:** S15 (Œî-new register node ‚Äî Plane-O container substrate). **Unlocks:** C12 (coverage-guided fuzzing maturity). **Enforced context budget:** seeds enter Bob only as *paths + sizes + hashes*, never contents; the brief/`repo-env.json` summary is capped (`MAX_SEED_DIRS = 16`) and the on-disk manifest is capped (`MAX_SEEDS = 2000`, per-dir cap), with a `truncated` flag when exceeded.

## What it does / holds
**As an index, it stores:** a deterministic, bounded manifest of where real fuzz inputs live in the target tree. For each discovered seed dir: `{ rel_path, file_count, total_bytes, has_zip, sample_rels[] (capped), truncated }`, plus a manifest-level `seed_corpus_hash` computed via `hashDocumentExcluding(doc, ["generated_at","seed_corpus_hash"])` (`fabric-common.js:99`) over the sorted `(rel, size_bytes)` tuples ‚Äî no full-byte reads, so it stays bounded and stable across re-runs.

**The bounded query it enables:** "for this C/C++ target, which directories hold real parser inputs, and what is the single `fuzz`-role command that stages `/src` into `/work/repo` and runs the harness seeded from them?" That command becomes a first-class entry in `recommended_commands[]` (`repo-env.js:507`), read by the evaluator's `fuzz_run` lens (`prompts/roles/evaluator.md:57`, `:64`).

**What it does NOT produce:** a working coverage-guided fuzzer. Plateau detection, the sanitizer matrix, structure-aware harness hints, and the aarch64 `compiler-rt`/libFuzzer-link fix are **C12's** scope (`reference_bob_oss_libcupsfilters_fuzz_image`), not IP8's. IP8 emits the role + the seeds; it must not claim the harness links.

## Failure mode it prevents
Two concrete bad outcomes:
1. **Blind fuzzing from empty seeds plateauing instantly.** Without a corpus the fuzzer burns its budget rediscovering file-format magic bytes instead of reaching deep parser states ‚Äî exactly the libde265 result on record (`project_libde265_oss_review_2026_05_31`: "blind fuzzing plateaus; only structure-aware HEVC fuzzing might yield"). Real `testdata/` seeds give the fuzzer a running start into the state machine.
2. **A declared-but-dead capability (the silent-drop class).** `fuzz` sits in `RECOMMENDED_COMMAND_ROLES` (`repo-env.js:141`) and in the evaluator lens, advertising a capability the emitter never produces ‚Äî the same registry-vs-code drift S12 exists to outlaw. Emitting the role makes the declaration honest.

## Context budget
IP8 introduces **no subagent and no new brief slice**. Its only context surfaces are: (1) the `fuzz`-role entry in `recommended_commands[]` ‚Äî a single command object; (2) the `seed_corpus` summary in `repo-env.json`/`repo-inventory.json` ‚Äî capped at `MAX_SEED_DIRS = 16` one-line entries (‚âà ‚â§1k tokens), with the full list (`MAX_SEEDS = 2000`, per-dir capped) held on disk and dereferenced by tool, not inlined. The evaluator reads `repo-env.json` directly (`evaluator.md:64`), so the cap lives on the emitted command + manifest, not on a separately-budgeted brief.

## Risks & doctrine compliance
- **index-without-consumer:** Guarded but flagged. The Œî1-lite half has an *immediate* consumer ‚Äî the emitted `fuzz` command references the seed dirs, read by the evaluator's `fuzz_run` lens today. The Œî3 seed-store's *full* value, however, is gated on C12; if C12 slips, the manifest must not become an orphan. Mitigation: ship slice 4 (emit role) so the command itself is the consumer regardless of C12 status; do not ship the heavy ingest without at least that emission.
- **substrate-as-trajectory:** N/A ‚Äî the seed store is a queried substrate index, not a step/trajectory log.
- **context-as-strategy:** Avoided ‚Äî bounded summary, never raw seed dumps into context.
- **auto-kill / freeze-then-verify / no-premature-termination:** IP8 only *adds* seeds; it has no kill gate. "No seeds found" must never drop a surface or terminate a hunt ‚Äî absence just means blind fuzzing, which C12's plateau detection (not IP8) reasons about. This preserves no-premature-termination.
- **Plane-O O-P1 (`--network none`, init-never-clones):** Seed discovery is a pure local fs walk of the already-bound `/src` ‚Äî no clone, no fetch, no network. The emitted fuzz command runs in the `--network none`, `--cap-drop ALL`, `/work`-staged container (O-P3, `evaluator.md:59`).
- **Hard-rule (no-PII) / O-P7:** Only seed *paths/sizes/hashes* enter the manifest and brief ‚Äî **never seed contents**. Seed bytes are attacker-shaped test inputs; they touch Bob only inside the `--network none` container during an actual fuzz run, never the LLM context. Because no untrusted bytes reach a brief, IP8 needs no S13/X6 datamarking envelope of its own (that boundary is the fuzz *contents*, which IP8 deliberately never ingests). The O-P7 scrub still runs over filenames.
- **Honesty constraint:** IP8 does not fix the aarch64 `compiler-rt` libFuzzer-link gap (`feedback_bob_oss_aarch64_libfuzzer_harness`); that is explicitly C12. IP8 emits a role and a corpus; it must not advertise a linking, coverage-guided fuzzer.
