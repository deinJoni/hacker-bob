## What it does / holds
S15 is a **register**: it gives the OSS evaluation substrate a first-class slab identity and pins its invariants so downstream nodes can name a real predecessor instead of the wrong one (S6 = `bob_import_static_artifact`, `docs/capability-hypergraph.md:51`). The substrate it registers, with the exact invariants it holds:

- **O-P1 / local-repo-first, never-clones.** `bob_init_repo_session` binds a session to an *already-checked-out* directory; it never fetches. Enforced by `governance-contracts.assertRepoRootPath` (`mcp/lib/governance-contracts.js:48-84`): the path must be absolute, `realpathSync`-resolvable, and a directory, or it throws `repo_path_not_found`/`repo_path_not_directory`. `source_url` is "Stored for provenance; never used to clone" (`init-repo-session.js:38`); `repo-target.js:9-11` states "No clone-from-remote happens here." Tool meta `network_access: false` (`init-repo-session.js:65`).
- **`--network none` by default, offline by construction.** Runtime argv: `args.push("--network", allowNetwork ? "bridge" : "none")` (`mcp/lib/repo-env.js:838`); build argv mirrors it (`repo-env.js:407`). Opening the network requires explicit `allow_network: true` and pins `--dns 1.1.1.1` to bypass host resolvers.
- **read-only `/src` + session-writable `/work` staging.** Mounts: `${repoRoot}:/src:${mountSuffix}` (default `ro`) and `${workDir}:/work:rw` (`repo-env.js:852-856`); mount-mode enum is exactly `["read_only","read_write"]` (`repo-env.js:763`). The image pre-stages `/work/repo` and `/work/out`, chowned `1000:1000`, `WORKDIR /src` (`repo-env.js:384-388`). This is the writable plane S14's in-container `git checkout` is required to use *instead of* re-enabling clone in init.
- **O-P3 sandbox floor.** `--cap-drop ALL`, `--security-opt no-new-privileges`, `--user 1000:1000`, `--cpus 2`, `--memory 4g`, `--pids-limit 1024`, `--read-only`, `--tmpfs /tmp:size=512m` (`repo-env.js:844-851`).
- **Default-no-build.** Startup writes only the plan + Dockerfile; an image build requires explicit `build_image: true, allow_network: true` (`docs/OSS_MODE_MVP.md:94-98`).
- **Role separation.** `bob_repo_docker_run` runs under `["evaluator-shared","verifier","evidence"]` — "Orchestrator dispatches but does NOT execute docker" (`repo-docker-run.js:86-88`); `init`/`prepare_env` are orchestrator-only (`repo-prepare-env.js:70`).

There is no "finding a triager recognizes" here — register is a slab node, so its product is the slab-table entry plus the enforced invariant set that every OSS-cluster node now correctly depends on.

## Failure mode it prevents
**Systemic predecessor mislabel that hides policy contracts.** Without an S-node, five OSS-cluster nodes (S14/I9/I10/IP7/IP8) name S6 (static-artifact import) as predecessor — a node that does *not* provide the container, the `--network none` default, or the `/work` mount. That mislabel is exactly what let S14's original anchor propose an `init`-clone flag that violates O-P1 (caught in WEAVE Must-fix #3). Registering S15 makes the real substrate — and its three load-bearing policy invariants (no-clone, network-none, read-only-/src) — a named contract that any downstream anchor is checked against, instead of an implicit assumption scattered across `repo-env.js`/`repo-target.js` comments.
