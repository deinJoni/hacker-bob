# I11 — Calibration ledger / trust cells
**Kind / Tier / Type:** Index (I) / Δ4 / adopt   **Verdict from verification:** **keep** (WEAVE roll-up: "Clean non-binding ledger — but source labels from grade verdicts + adjudication, NOT the I6 `calibration_label` slot (now deleted, verified)"). ## Spec (one-paragraph contract)
**Inputs:** per-finding outcome labels emitted at two existing decision points — grade verdicts (`mcp/lib/grade-verdict-store.js` `writeGradeVerdict`, :311) and verification adjudication (`mcp/lib/verification.js`, disposition ∈ `["confirmed","denied","downgraded"]` per `constants.js:58`), each tagged with a `model` id and an `attack_class`. **Outputs:** (a) an append-only per-session calibration ledger of `(model, attack_class, decision_class, outcome)` rows; (b) a new cross-session read tool that computes, per cell, count + observed success-rate + **Wilson score-interval lower bound**, returned as a **NON-BINDING** prior. **Predecessors:** S11 (Δ-new cross-session label/metric store). **Unlocks:** grader prior (augment only) and X7b (cost-per-finding). **Enforced context budget:** the prior slice handed to the grader is capped at ≤2k tokens (matching I6's deleted `priors_slice` budget, `capability-hypergraph.md:218`); the cross-session scan is bounded by the existing `CROSS_SESSION_ANALYTICS_MAX_SESSIONS = 200` cap (`pipeline-analytics.js:41`); the on-disk read is capped exactly like `pack-telemetry.js` (`RECORD_BYTE_CAP = 64 MiB`, :57).

## What it does / holds
It **stores** one JSONL row per realized decision: `{model, attack_class, decision_class, outcome, finding_id_hash, ts}` where `decision_class ∈ {grade, adjudication_disposition}`, and `outcome` is the realized label (`grade`→ verdict bucket from `GRADE_VERDICT_VALUES`; `adjudication_disposition`→ `confirmed|denied|downgraded`). No finding prose, no PII, no secrets — only typed labels and a hash of the finding id.

The **bounded query** it enables: given `(model, attack_class, decision_class)`, return `n`, `successes`, point estimate `p̂`, and the **Wilson lower bound** `p_lo` at a fixed `z` (default 1.96). The Wilson lower bound is the deliberate choice over raw `p̂`: at low `n` it stays conservative (a cell with 1/1 confirmed reports a low bound, not 100%), which is exactly the property a non-binding prior needs so a thin cell never looks authoritative. The aggregate is fed to the **grader only** (sanitized rubric distribution, per `X-rej-3`) to nudge `severity_accuracy`/`proof_quality` scoring — it never enters a hunt brief and never gates a finding.

## Failure mode it prevents
Bob having **no memory of its own calibration**: repeatedly over-scoring an `(model, attack_class)` combination the grader's own later adjudication kept marking `denied`, with zero feedback signal. The deleted I6 `calibration_label` slot was the only place this ground truth used to flow; after `c02179c` it is dropped on the floor. I11 reconnects realized outcomes to future grading **without** reintroducing the anti-pattern that doctrine forbids — i.e. it must not become a pre-verification kill switch (see Risks).

I11 is correctly the deepest tier; it must not be pulled forward to gate any Δ1–Δ3 capability.

## REVIEW (parishioner / real-target gate)
**Honest gate = ENGINEERING/FIXTURE, long-horizon** (per the node's `gate` field and WEAVE must-fix #11: "calibration mostly dormant at Bob's claim volume"). The closing criteria are: (1) the ledger persists labels at both decision points across ≥2 real sessions; (2) `bob_read_calibration` returns a correct Wilson bound over a **seeded multi-session fixture corpus**; (3) the grader-augment path measurably shifts a score on a replayed session **without** changing the reportable set. There is **no** real-target finding this node produces on its own — it is observability/prior infrastructure that only pays off after many sessions accumulate per-cell `n`. State this explicitly so the Δ4 gate is not mistaken for a findings gate.

## Context budget
- Grader prior slice injected into the grade brief: **≤2k tokens** (hard).
- Per-cell summary line: **≤200 tokens**.
- `bob_read_calibration` response: cap returned cells (e.g. top-N by `n`, default ≤50) so a large corpus cannot blow the orchestrator read.
- On-disk read guard: `RECORD_BYTE_CAP = 64 MiB`; cross-session fan-out ≤200 sessions.
- **No new subagent is introduced** — there is no per-cell or per-finding triage agent, so no subagent token cap applies; the only context surface is the bounded read + the ≤2k grader slice.

## Risks & doctrine compliance
- **auto-kill that inverts freeze-then-verify / no-premature-termination (PRIMARY risk):** a calibration prior is one refactor away from "this cell's `p_lo` is low, so suppress/skip the finding." That is exactly `X-rej-1` (suppress 0-days) and `X-rej-2` (FP prefilter before the verifier). **Hard contract:** the prior is `NON-BINDING`, feeds the **grader only** (`X-rej-3` — never the hunt brief), and the AV of consumption is *score nudging*, never gating, pruning, or short-circuiting verification/adjudication. The read tool is `mutating:false` and has no dispatch authority. Encode in code + tests that no calibration value can remove a finding from the reportable set.
- **index-without-consumer:** mitigated by the explicit `grader prior (augment)` + `X7b` unlocks; but the augment consumer is a *follow-up commit*, so until it lands the ledger is a producer-without-consumer. Acceptable only because X7b independently consumes the same store — do not let the ledger sit orphaned indefinitely (this is the I9/I6 silent-drop failure mode the whole plane exists to prevent; register its consumed fields if S12's `stigmergic-consumers.js` registration is in play).
- **substrate-as-trajectory:** avoided — the ledger stores realized labels (a real bounded-query index), not raw run traces; it reuses S11's existing cross-session substrate rather than minting a new store.
- **context-as-strategy:** avoided — the prior is a ≤2k sanitized distribution, not "stuff all past outcomes into the grade brief."
- **Policy / hard-rule compliance:** labels carry no operator PII and no target identifiers (`index_only_no_target_url_export`); all rows pass `validateNoSensitiveMaterial`. Plane-O O-P1 (`--network none`) is not directly engaged (this is grade/adjudication-time local file I/O, no container), but the tool is `network_access:false` and reads only local session dirs.
- **(assumed) — two unverifiable-in-code dimensions:** `model` has **no** native MCP source (grep-confirmed) and `attack_class` has **no** claim field (`claims.js` confirmed). Both are flagged as supplied-or-degrade-to-`"unknown"`. If the grader cannot reliably supply `model`, the ledger collapses to a single model bucket and loses its headline per-model discriminating power — this is the same "Bob lacks native model/token telemetry" limitation X7b is built around (`project_bob_oss_benchmark_plan`: reconstructable from CC transcripts, not natively recorded). Resolve the model-id provenance before treating per-`model` cells as trustworthy.
